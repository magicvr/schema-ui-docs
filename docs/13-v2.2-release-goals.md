# v2.2 发布目标与门禁

本文档记录 **v2.1 → v2.2** 的 MINOR 发布目标：表格当前页多选与批量 request（ADR-0022）以 **capability 门控** 进入核心协议。未声明新 capability 的合法 `2.1` 页面语义不变。

权威 ADR：[0022](./decisions/0022-table-selection-and-batch-request.md)。迁移见 [migrations/2.1-to-2.2.md](./migrations/2.1-to-2.2.md)。轨道见 [11-next-admin-lifecycle-goals.md](./11-next-admin-lifecycle-goals.md)。

## 1. 版本策略（V275 裁决）

1. **语义已在树中：** Schema / L2 / fixtures / reference 已支持 `table.selection` 与 `actions.batch.request`；在 `2.2.0` tag 前，页面可声明 `protocolVersion: "2.1"` + 上述 capability。
2. **正式发布时：** 制品与页面协议升至 `2.2.0` / `"2.2"`；使用本 MINOR 字段的官方样例与相关算法 fixtures **统一** `protocolVersion: "2.2"`。
3. **发布之后：** 使用 2.2 字段的页面 **必须** 声明 `"2.2"`（与 2.1 字段要求 `"2.1"` 同纪律）；纯 2.1 页面可继续 `"2.1"`。
4. **不建议**在 V273/V274 类互操作缺口未闭合时宣称 2.2 语义冻结；本文件门禁关闭后方可打 `v2.2.0` tag。

## 2. v2.2 阻断门禁

- [x] ADR-0022 `accepted`；capability 键写入 `08` 预定义表。

- [x] L2 字段集→`protocolVersion` 下限（V282）；过渡期 `ALLOW_22_FIELDS_ON_21=true`。
- [ ] 打 tag 时：`ALLOW_22_FIELDS_ON_21=false`；`package.json` / lockfile / `protocol-manifest.json` 对齐 `2.2.0` / `2.2`。
