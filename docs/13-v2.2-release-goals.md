# v2.2 发布目标与门禁

本文档记录 **v2.1 → v2.2** 的 MINOR 发布：表格当前页多选与批量 request（ADR-0022）以 **capability 门控** 进入核心协议。未声明新 capability 的合法 `2.1` 页面语义不变；使用 2.2 字段的页面必须声明 `meta.protocolVersion: "2.2"`。

权威 ADR：[0022](./decisions/0022-table-selection-and-batch-request.md)。迁移见 [migrations/2.1-to-2.2.md](./migrations/2.1-to-2.2.md)。轨道见 [11-next-admin-lifecycle-goals.md](./11-next-admin-lifecycle-goals.md)。

## 1. v2.1 到 v2.2 的边界

- v2.2 页面声明 `meta.protocolVersion: "2.2"`。
- Renderer `supportedVersions` 可同时包含 `"2.1"` 与 `"2.2"`。
- 新增预定义 capability：`table.selection`、`actions.batch.request`（既有 2.1 Admin P0 能力不变）。
- 新增字段：`table.props.selection`、`ActionTrigger.requiresSelection`、`batchMapping`、`$selection.keys` / `$selection.count`（仅 batchMapping）。
- **L2 字段集下限（V282）：** 2.2 字段要求 `protocolVersion >= "2.2"`（`ALLOW_22_FIELDS_ON_21=false`）。
- 不引入跨页全选、部分成功、权限继承（仍属后续轨道）。

## 3. 发布工程门禁

- [x] `package.json` / lockfile / `protocol-manifest.json` 统一为 `2.2.0` / `2.2`。
- [x] 算法类 fixtures `protocolVersion` 为 `"2.2"`（version-negotiation 历史向量保留）。

- [x] L2 `ALLOW_22_FIELDS_ON_21=false`。

## 4. 版本纪律

v2.2 之后继续遵守 SemVer：文档勘误为 PATCH；兼容字段 / capability 为 MINOR；破坏合法输入或默认可观测结果为 MAJOR。权限继承等后续能力另开 MINOR/ADR，不回写本文件门禁。

## 5. 完成定义

G1–G4 与发布工程门禁全部关闭，且 `npm run release:check` 与 `npm run verify:protocol-artifact` 通过，即可将 `2.2.0` 视为协议制品完成。v2.1 历史门禁保留在 [12-v2.1-release-goals.md](./12-v2.1-release-goals.md)。
