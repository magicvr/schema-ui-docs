# v2.2 到 v2.3 迁移指南

v2.3 是 **向后兼容的 MINOR** 发布：新增容器 `edit` / `delete` 权限继承与操作入口的显式意图（ADR-0023）。页面仅在声明 `permissionCascade` 或 `permissionIntent` 时才进入新语义；未声明这些字段及 `permissions.inheritance` 的 v2.2 页面继续只按已有本地 `permissions` 求值。

权威 ADR：[0023](../decisions/0023-container-permission-inheritance.md)。发布门禁见 [14-v2.3-release-goals.md](../14-v2.3-release-goals.md)。

## 1. 版本声明与协商

```yaml
meta:
  protocolVersion: "2.3"
  requiredCapabilities:
    - permissions.inheritance
```

| 页面字段 | 所需 capability | `protocolVersion` |
|---|---|---|
| Node `permissionCascade` | `permissions.inheritance` | `"2.3"` |
| RowAction `permissionIntent` | `permissions.inheritance` | `"2.3"` |
| toolbar / `actionButton.props` `permissionIntent` | `permissions.inheritance`（以及各入口原有 capability） | `"2.3"` |

- Renderer `supportedVersions` 应包含 `"2.3"`，并可继续支持 `"2.2"`。
- 不支持 `permissions.inheritance` 的 Renderer 遇到声明该 capability 的页面必须在加载前拒绝，不能部分降级。
- **L2 字段集下限：** `permissionCascade` / `permissionIntent` 要求 `protocolVersion >= "2.3"`；只写 capability 不足以把字段挂在 `"2.2"` 页面上。

## 2. 从隐式约定迁移到显式边界

不要把 v2.2 容器上的既有 `permissions.edit` / `permissions.delete` 当作自动继承。需要收紧后代时，增加同名 source 和明确 `permissionCascade.keys`：

```yaml
type: section
permissions:
  edit: "$context.user.roles contains 'editor'"
  delete: "$context.user.roles contains 'admin'"
permissionCascade:
  keys: [edit, delete]
children:
  - type: form
    props:
      submitAction: saveOrder
    children:
      - type: input
        props: { field: customerName, label: 客户名称 }
```

- 可声明 cascade 的 Node 只有 `section`、`grid`、`form`、`tabs`、`table`；keys 非空、去重且仅为 `edit` / `delete`。
- default form 中的 `input`、`inputNumber`、`datePicker`、`dateRangePicker`、`select`、`upload` 以及 `submitAction` 使用 effective `edit`；search form 不参与。
- 有效权限是祖先 cascade source 与目标本地 `permissions` 的 AND。子节点只能继续收紧，不能重新授权。
- modal `content` 和 navigated page 是新根；不要期望触发入口的权限边界自动穿透。

## 3. 操作入口迁移

需要让 RowAction、toolbar Trigger 或 actionButton 参与继承时，显式写出意图：

```yaml
props:
  actions:
    - key: delete
      label: 删除
      permissionIntent: delete
  toolbar:
    - key: deleteAll
      label: 批量删除
      actionRef: deleteSelected
      permissionIntent: delete
```

- `permissionIntent` 仅允许 `edit` / `delete`，且只能写在 `table.props.actions[]`、`table.props.toolbar[]` 或 `actionButton.props`。
- form default submit 是隐式 `edit`，不得另写 `permissionIntent`；顶层 `actions` 是定义，不是入口。
- **`table.props.columns[]` 不参与权限级联。** 列保留已有本地 `permissions`，不允许 `permissionIntent`。未标注意图的操作入口也只适用本地权限。
- 不得用 `key`、`actionRef`、HTTP method、URL 或文案猜测 edit/delete 意图。

## 4. 执行行为

对于带 intent 的操作入口和 default form submit，执行顺序固定为：`visibleWhen` → effective/local permission → `disabled` OR `requiresSelection` → stop → `confirm` → action。任何拒绝都必须发生在 confirm、请求构造、导航、打开 modal 或提交之前。隐藏还是禁用仍由组件既有呈现契约决定，但不可执行性不可降级。
