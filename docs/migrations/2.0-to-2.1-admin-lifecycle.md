# 2.0 → Admin 生命周期 P0（ADR-0020 / 0021）迁移说明

本说明描述 **已接受** 的页面级动作入口与记录导航/回填能力。完整制品版本号升至 `2.1.0` / `protocolVersion: "2.1"` 前，页面可继续使用 `protocolVersion: "2.0"`，但必须通过 `meta.requiredCapabilities` 声明新能力；不支持这些 capability 的 Renderer 必须在加载前拒绝页面。

## 1. 新增 capability

| capability | 何时需要 |
|---|---|
| `actions.page.trigger` | 使用 `actionButton` 或 `table.props.toolbar` |
| `actions.row.navigate` | `RowAction.actionRef` 指向 `type: navigate` |
| `form.record.load` | 使用 `form.props.recordSource` |

既有 `actions.row.request` / `actions.upload` 不变。行级 `actionRef` 指向 `request` 时仍只要求 `actions.row.request`。

## 2. 对既有 2.0 页面的影响

- **无新字段的页面：** 行为不变，无需修改。
- **仅 Host 私有工具栏/回填：** 可逐步改为声明式字段；在改完前继续用 Host Extension，但不得声称属于核心互操作能力。

## 3. 配置迁移对照

| 旧（Host / 习惯） | 新（协议） |
|---|---|
| 壳子上硬编码「新建」 | `table.toolbar` 或 `actionButton` + `navigate`/`modal` |
| 行「编辑」本地 handler 拼 query | `actionRef` → `navigate` + `navigateMapping` |
| 编辑页 JS 拉详情后 `setFields` | `form.recordSource` + 必填 `responseMapping` |
| 隐藏字段塞 id | 只读可见字段 + mapping（暂无 `type: hidden`） |

## 4. 静态校验要点

- Trigger `key` 必填；toolbar 内 key 唯一。
- 页面级 request 禁止 GET。
- `requestMapping` 与 `navigateMapping` 互斥；类型与 mapping 必须匹配。
- `recordSource` 禁止出现在 `mode: search`；`responseMapping` 必填非空；url 仅内联。
