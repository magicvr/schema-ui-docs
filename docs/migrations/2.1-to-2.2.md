# v2.1 到 v2.2 迁移指南

v2.2 是 **向后兼容的 MINOR** 发布目标：新增表格**当前页多选**与工具栏**批量 request**（ADR-0022）。仅使用 v2.1 字段集的页面可继续声明 `protocolVersion: "2.1"`；若页面使用 `table.props.selection`、`ActionTrigger.requiresSelection` 或 `batchMapping`，在 **`2.2.0` 制品正式发布后必须** 声明 `protocolVersion: "2.2"` 与对应 `requiredCapabilities`。

> **发布前过渡（V275）：** 在 `schema-ui-protocol-2.2.0` tag 之前，合法页面可继续用 `protocolVersion: "2.1"` + `table.selection` / `actions.batch.request` 依赖 capability 协商。正式发布后的纪律见下表与 [13-v2.2-release-goals.md](../13-v2.2-release-goals.md)。

权威 ADR：[0022](../decisions/0022-table-selection-and-batch-request.md)。轨道背景见 [11-next-admin-lifecycle-goals.md](../11-next-admin-lifecycle-goals.md)。

## 1. 版本声明与协商

```yaml
meta:
  protocolVersion: "2.2"   # 2.2.0 发布后；发布前过渡期可用 "2.1"
  requiredCapabilities:
    - actions.page.trigger     # toolbar / actionButton
    - table.selection          # 若启用多选
    - actions.batch.request    # 若使用 batchMapping
```

| 页面字段 | 所需 capability | 发布后 `protocolVersion` |
|---|---|---|
| `table.props.selection` | `table.selection` | `"2.2"` |
| toolbar `requiresSelection` | `table.selection`（及通常的 `actions.page.trigger`） | `"2.2"` |
| toolbar `batchMapping` | `actions.batch.request`（及通常 `table.selection` + `actions.page.trigger`） | `"2.2"` |

- Renderer `supportedVersions` 应包含 `"2.2"`（可同时支持 `"2.1"` / `"2.0"`）。
- 仅声明 `"2.1"` 的 Renderer 遇到 `protocolVersion: "2.2"` 页面时必须 `UNSUPPORTED_PROTOCOL_VERSION`。
- 标准入口不做 2.1→2.2 猜测；无自动 legacy adapter。
- **L2 字段集下限（审计 0064 / V282）：** `2.2.0` 发布前 L2 `ALLOW_22_FIELDS_ON_21=true`，2.2 字段可挂 `"2.1"`；tag 时改为 `false`，此后 2.2 字段要求 `protocolVersion >= "2.2"`。

## 2. 不变的 2.1 边界（提醒）

下列规则 **未** 在 2.2 放宽：

- 页面 Trigger / 行级 request / recordSource 的既有语义（ADR-0020 / 0021）。
- DataRef 只读 GET、query 序列化、保留参数 `page` / `pageSize` / `sort`。
- 跨页全选、部分成功、权限继承 **仍不在** 本 MINOR。

## 3. 新能力迁移对照

| 旧做法（Host / 私货） | v2.2 协议 |
|---|---|
| 本地 `selectedRowKeys` + 硬编码批量 API | `table.props.selection.mode: multiple` + toolbar `batchMapping` |
| 空选仍发请求 / 各端 count 语义不同 | 规范化后 `count === keys.length`；`EMPTY_SELECTION`；可选 `requiresSelection: true` |
| 翻页后仍保留跨页勾选 | **不支持**；筛选/翻页/排序/reload **清空**选中 |
| body 塞整行对象数组 | 仅 `$selection.keys`（body）与可选 `$selection.count` |

### 3.1 选择模型

- MVP 仅 `mode: multiple`。
- 键 = 当前行 `rowKey` 标量（string / finite number / boolean）；非标量静默丢弃；去重保序。
- 不写入 URL query。

### 3.2 批量 request

- 仅 `table.toolbar` + `type: request`；method：POST/PUT/PATCH/DELETE（禁止 GET）。
- `$selection.keys` **仅 body**；path 仅字面量且与 url `{name}` 双向对齐。
- 构造入口与 table-query-state **同一** selection 规范化（V274 / V281）。
- 一次点击 = 一次 HTTP；成功 `reload` 清空选中。

## 4. 官方场景与 conformance

- 扩展示例：[`05-scenarios/admin-list-batch.md`](../05-scenarios/admin-list-batch.md)、[`_samples/order-list-batch.yaml`](../05-scenarios/_samples/order-list-batch.yaml)。
- fixtures：`request-construction`（batch*）、`search-table` / table-query-state selection 清空、`version-negotiation` capability 向量。
- **`2.2.0` 打包时**：使用 2.2 字段的官方/扩展示例与相关算法 fixtures 的 `protocolVersion` 升至 `"2.2"`（见发布目标 G3）。

## 5. 工具与制品

- 正式制品：`schema-ui-protocol-2.2.0.tar.gz`（tag / SHA-256 在发布时固定）。
- 验证器 / MCP 若捆绑协议，应更新其 `schemaUiProtocol.artifactVersion`（工具自身 SemVer 可独立）。
