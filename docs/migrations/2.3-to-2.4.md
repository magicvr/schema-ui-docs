# v2.3 到 v2.4 迁移指南

v2.4 是 **向后兼容的 MINOR** 发布：新增标准只读详情组件 `recordView`（ADR-0024）。页面仅在使用 `type: recordView` 时才进入新语义；未使用该组件的合法 `2.3` 页面行为完全不变。

权威 ADR：[0024](../decisions/0024-record-view.md)。发布门禁见 [15-v2.4-release-goals.md](../15-v2.4-release-goals.md)。

## 1. 版本声明与协商

```yaml
meta:
  protocolVersion: "2.4"
  requiredCapabilities:
    - record.view.load
```

| 页面字段 | 所需 capability | `protocolVersion` |
|---|---|---|
| `type: recordView` | `record.view.load` | `"2.4"` |

- Renderer `supportedVersions` 应包含 `"2.4"`，并可继续支持 `"2.3"`。
- 不支持 `record.view.load` 的 Renderer 遇到声明该 capability 的页面必须在加载前拒绝，不能部分降级。
- **L2 字段集下限：** `recordView` 要求 `protocolVersion >= "2.4"`；只写 capability 不足以把组件挂在 `"2.3"` 页面上。
- `legacy adapter`：旧 MAJOR 页面仍须在入口前显式转换；本 MINOR 不改变 adapter 边界。
- 既有 `query` / `pageSize` / `requestMapping` / `actions.upload` 语义不变。

## 2. 从凑合只读页迁移到 recordView

**不要** 再把禁用 form 或一堆 `text`/`statCard` 当作跨实现正例详情页。推荐：

```yaml
type: recordView
props:
  title: 基本信息
  recordSource:
    method: GET
    url: /api/orders/{orderId}
    path:
      orderId: $context.route.query.orderId
    responseMapping:
      orderId: orderId
      customerName: customer.name
      status: status
      amount: amount
  fields:
    - { key: orderId, label: 订单号 }
    - { key: customerName, label: 客户 }
    - key: status
      label: 状态
      format: tag
      tagMap:
        paid: { text: 已支付, tone: success }
    - { key: amount, label: 金额, format: currency }
```

规则摘要：

- `recordSource` 与 `form.recordSource` **同构**（GET、path/query 仅 `$context.route.*` 或字面量、显式 `responseMapping`）。
- `fields[].key` 必须是 `responseMapping` 的键；mapping 可含未展示的多余键。
- `format: tag` 时 `tagMap` 必填。
- 无 `submitAction`、无 children、不用 DataRef 承载主记录。
- 列表进详情继续用既有 `actions.row.navigate`；返回/编辑用同页 `actionButton`（`actions.page.trigger`）。

## 3. 与 form.record.load 的边界

| 场景 | 使用 |
|---|---|
| 编辑回填并提交 | `form` + `recordSource` + `form.record.load` |
| 只读查阅 | `recordView` + `record.view.load` |

两者 capability **独立**；可同时声明。不要用只读 form 伪装详情作为官方正例。

## 4. 执行行为

1. 挂载后构造 GET（与 request-construction `recordSource` 向量一致）。
2. 映射成功 → `values`；缺失路径 → 该键 `null`。
3. 失败 → 节点 error 态；401/403 走全局认证。
4. 未声明 `recordView` 的 v2.3 页面：本地 `permissions`、批量、继承等语义 **不变**。
