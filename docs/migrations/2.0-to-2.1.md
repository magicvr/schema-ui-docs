# v2.0 到 v2.1 迁移指南

v2.1 是 **向后兼容的 MINOR** 发布：新增 Admin 生命周期 P0 字段与 capability。仅使用 v2.0 字段集的页面可继续在声明 `protocolVersion: "2.0"` 的 Renderer 上运行；若页面使用任一 v2.1 新字段，必须声明 `protocolVersion: "2.1"` 与对应 `requiredCapabilities`。

从更早版本迁到 2.x 时，请先完成 [1.0-to-2.0.md](./1.0-to-2.0.md)（含 **legacy adapter**、DataRef **query**、保留 **pageSize**、行级 **requestMapping**、**actions.upload** 等边界），再应用本文件。

## 1. 版本声明与协商

```yaml
meta:
  protocolVersion: "2.1"
  requiredCapabilities:
    - actions.page.trigger   # 若使用 toolbar / actionButton
    - actions.row.navigate   # 若行级 navigate
    - form.record.load       # 若 form.recordSource
```

- Renderer `supportedVersions` 应包含 `"2.1"`（可同时支持 `"2.0"`）。
- 仅声明 `"2.0"` 的 Renderer 遇到 `protocolVersion: "2.1"` 页面时必须 `UNSUPPORTED_PROTOCOL_VERSION`。
- 标准入口不做 2.0→2.1 猜测；无自动 **legacy adapter** 串联（与 v2 严格协商一致）。
- **L2 字段集下限（审计 0064 / V282）：** 使用 `actionButton` / `table.toolbar` / `navigateMapping` / `form.recordSource` 时，页面 `protocolVersion` 必须 ≥ `"2.1"`（不得仍为 `"2.0"` + capability）。

## 2. 不变的 2.0 边界（提醒）

下列规则 **未** 在 2.1 放宽，迁移时不要回退：

- DataRef / 页面 API 只读 **GET**；**query** 标量序列化与 ADR-0010 不变。
- 保留 query 名 `page` / **pageSize** / `sort`（ADR-0011）不变。
- 行级 **requestMapping** 标量 / own-property 规则不变；`actions.row.request` 语义不变。
- **actions.upload** 与串行多文件规则不变。
- URL 仍为 baseURL 下单斜杠相对路径。

## 3. 新能力迁移对照

| 旧做法（Host / 私货） | v2.1 协议 |
|---|---|
| 壳子硬编码「新建」 | `table.props.toolbar` 或 `actionButton` + `actions.page.trigger` |
| 行「编辑」本地拼 URL | `actionRef` → `type: navigate` + `navigateMapping` + `actions.row.navigate` |
| 编辑页 JS 拉详情 setFields | `form.props.recordSource` + 必填 `responseMapping` + `form.record.load` |
| 假隐藏字段塞 id | 只读可见字段承载主键/version（暂无 `type: hidden`） |

### 3.1 页面级 Trigger

- `key` 必填；`actionRef` 仅 `request` \| `navigate` \| `modal`。
- Trigger 触发的 `request` **禁止 GET**。

### 3.2 行级 navigate

- 与 `requestMapping` 互斥；仅 `path` / `query`。
- 值仅字面量或 `$row.*`。

### 3.3 记录加载

- `method: GET`；url **仅内联**（不得 ref datasources）。
- `responseMapping` **必填非空**。
- `mode: search` 禁止 `recordSource`。
- 宿主注入 `$context.route.query|params` 只读快照。

## 4. 官方场景与 conformance

- 官方六场景示例已使用 `protocolVersion: "2.1"`。
- Admin 闭环扩展示例与 fixtures：`admin-list-edit-lifecycle`、`request-construction`（`rowNavigate` / `recordSource` / `pageTriggerRequest`）、`formRecord` 映射。

## 5. 工具与制品

- 固定协议制品 `schema-ui-protocol-2.1.0.tar.gz` 的 tag / SHA-256。
- 验证器 / MCP 若捆绑协议，应更新其 `schemaUiProtocol.artifactVersion` 声明（工具自身 SemVer 可独立）。
