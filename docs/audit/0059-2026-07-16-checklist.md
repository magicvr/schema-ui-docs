## V236（P1）DataRef 只读边界

- [x] 决定 DataRef 只允许 `GET`，`params` 永远只进入 query；body-based read 不通过旧字段隐式表达。
- [x] 同步 [01-node-protocol.md](../01-node-protocol.md)、[04-datasource-contract.md](../04-datasource-contract.md)、[08-renderer-spec.md](../08-renderer-spec.md)、Node/Page Schema 和 [ADR-0013](../decisions/0013-dataref-read-only.md)。
- [x] L2 同时拒绝节点级 `data.source: api` 和页面级 `datasources.*` 的显式非 GET。
- [x] JS/Python request reference 对非 GET 返回 `DATA_REF_METHOD_NOT_READ_ONLY`。
- [x] request fixtures 覆盖非 GET DataRef；六个官方场景的合法 DataRef 仍通过。

## V237（P1）Action 重试与未知结果

- [x] Schema 和正文增加 `retryPolicy: never | idempotent`，默认 `never`。
- [x] [ADR-0014](../decisions/0014-action-replay-semantics.md) 定义 invocation id、`Idempotency-Key`、服务端去重、冲突、Abort 和 `unknown` 结果。
- [x] 明确 `never` 不自动重发；`idempotent` 在同一逻辑调用的重试中复用同一 key。
- [x] timeout/network outcome 标记为 `unknown`，HTTP 业务错误仍按既有 Action outcome 处理。
- [x] JS/Python request、upload、action outcome reference 和 fixtures 对齐；覆盖缺少 invocation id、非法策略、幂等头和未知结果。

## V238（P2）RowAction 标量和 own-property

- [x] 协议明确 `$row.*` 解析结果只能是 string、finite number、boolean、null；path/query/body 保持扁平 map。
- [x] JS reference 使用 own-property；Python reference 使用字典 own-key；缺失字段统一拒绝。
- [x] `__proto__`、`prototype`、`constructor` 路径统一拒绝，L2 与 reference 错误边界一致。
- [x] fixtures 覆盖合法 scalar、嵌套对象、数组、缺失字段和原型保留路径。
- [x] MCP 定向回归确认合法 RowAction 通过、非法 RowAction 被 L2 拒绝。

## V239（P2）DataRef body 语义

- [x] 选择 query-compatible read API 方案，不为 DataRef 增加隐式 body。
- [x] 明确 `data.params`、页面 datasource params 和 optionsSource params 的既有 query 序列化语义不变。
- [x] 明确需要 JSON body 的命令式请求必须使用 Action；未来 body-based read 必须新增独立 ADR、字段和 fixtures。
- [x] Schema、正文、ADR、L2 和 JS/Python reference 没有分叉解释。

## V240（P2）URL grammar

- [x] Page/Node/Action Schema、L2 和 Renderer 文档使用同一规则。
