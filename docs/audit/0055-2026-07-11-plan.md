# v1.0 推进实施计划

## 总体策略

先冻结运行语义，再建立共享 fixtures，最后切版本和发布。不要在 G2/G3 决策前批量改示例，也不要先把版本字符串改成 `1.0`；否则测试只会固化尚未决定的行为。

建议设两条并行工作流：

- **协议工作流：** G1 版本协商；G2 query ADR；G3 保留参数 ADR。
- **一致性工作流：** fixture 格式、JavaScript 参考执行器、后端消费者和 CI。

协议工作流完成 G2/G3 后，一致性工作流才能冻结请求类期望输出。

## 阶段 0：锁定基线（已完成）

**目标：** 让现有合法性能力成为独立、稳定的回归门禁。

**产物：**

1. 新增官方场景 YAML fence 公共提取器；
2. 新增 `validate:scenarios`，提取后执行 L0-L4；
3. CI 独立执行根场景校验、MCP build/test/tools smoke、Docker smoke；
4. 保存当前基线：6 个场景通过、MCP 117 个测试通过。

**退出条件：** PR 上能分别看到“协议合法性”和“MCP”两个绿灯；场景 Markdown 若不可提取或 YAML 非法，前者必须失败。

**完成证据：** `npm run validate:scenarios` 六场景 L0-L4 通过；MCP build、117 tests、tools smoke 通过；`schema-ui-mcp:0.3.0-rc.1` 本地构建与 Docker smoke 通过；CI/CD 已接入场景门禁。

## 阶段 1：关闭 G1

**目标：** 版本输入只有唯一接受/拒绝结果。

**产物：**

1. ADR-0009：严格版本与 capability 协商；
2. `08-renderer-spec.md` 删除最近版本猜测和隐式 `0.1`；
3. `conformance/fixtures/version-negotiation/*.json`；
4. JavaScript 参考函数 `negotiateProtocol(pageMeta, rendererSupport)`；
5. 显式 legacy adapter 接口说明，adapter 输出必须是带版本的标准页面。

**最小向量：** 支持 `0.3`、支持 `1.0`、未知 MINOR、未知 MAJOR、缺失版本、畸形版本、缺失 capability、未知 capability。

**退出条件：** Renderer 和参考 runner 对全部向量返回相同 `accept` 或结构化拒绝原因；标准入口不存在 fallback 分支。

## 阶段 2：关闭 G2 与 G3

**目标：** 任意合法页面状态产生唯一的最终 URL。

**决策建议：** 采用 RC 保守方案，只允许 query 标量。对象和数组留到 `1.x` 候选 ADR，不进入 v1.0。

**产物：**

1. ADR-0010：query 标量类型与字节级序列化；
2. ADR-0011：保留参数、来源优先级和状态重置；
3. 公共伪代码与 JavaScript 参考函数：`mergeQuerySources`、`serializeQuery`；
4. Schema/L2 拒绝对象、数组及保留 key 冲突；
5. DataRef、`optionsSource.params`、行级 Action 共用同一序列化入口；
6. 搜索提交、清空、翻页、排序状态转换 fixtures。

**建议算法：**

- `undefined`：省略；
- 字面量 `null`：省略；
- string：保留，包括空字符串；
- finite number：十进制 JSON 表示，拒绝 `NaN` / `Infinity`；
- boolean：小写 `true` / `false`；
- key/value：UTF-8 后按 RFC 3986 百分号编码，空格编码为 `%20`，不用 `+`；
- 已有 URL query 先解析为有序键值；协议 map 按 key 的 Unicode code point 升序；同名 key 由后层整体替换；最终不产生重复 key；
- fragment 不参与 query，序列化后原样附回；
- 合并顺序固定为：已有 URL query < 静态 params < 搜索字段 < Renderer 状态。

**退出条件：** JavaScript 参考实现与指定生产后端语言对全部 URL 字节向量一致；四类 query 来源没有各自的编码分支。

## 阶段 3：关闭 G4

**目标：** 从“配置合法”升级为“合法配置如何执行”。

**fixture 目录建议：**

```text
conformance/
  README.md
  schemas/
    fixture.schema.json
  fixtures/
    version-negotiation/
    request-construction/
    response-mapping/
    search-table/
    reactions/
    actions/
    errors/
    uploads/
    scenarios/
  reference-js/
  runner/
```

每个 fixture 至少包含 `id`、`protocolVersion`、`category`、`input`、`expected`。网络、时钟、随机数、宿主 hook 均以事件输入表示；期望输出只记录最终请求和有序事件，不记录框架内部状态。

**实施批次：**

1. 版本、请求构造、参数合并、`responseMapping`；
2. 搜索/分页/排序和六个场景；
3. reaction 快照、批量提交、冲突、循环保护；
4. Action、OutcomeBehavior、HTTP 错误、超时、中断；
5. 单文件/多文件上传与响应取值。

**跨实现要求：** 在阶段 2 开始时指定一个真实生产后端实现负责人和语言。该实现必须直接消费版本化 fixtures；若后端仓库独立，则 CI 下载由本仓库 commit SHA 固定的 fixture artifact，并回报结果。仅用第二个 JavaScript runner 不能满足后端一致性验收。

**退出条件：** 一个前端 Renderer、JavaScript reference 和一个生产后端消费者对共享 fixtures 全绿，且无 skip、allowlist 或私有期望文件。

## 阶段 4：RC 验证

**目标：** 在改版本前证明 v1 契约已经稳定。

1. 发布 `1.0.0-rc.1` 仅用于制品验证，页面协议仍按最终决策切为 `"1.0"`；
2. 冻结 fixtures，后续只允许修复错误期望并附审计记录；
3. 完整运行 L0-L4、MCP、tools smoke、Docker smoke、前端 Renderer 和后端一致性；
4. 对六个场景执行端到端期望验证；
5. 至少完成一次干净环境安装与固定 SHA fixture 消费。

**退出条件：** 连续两个候选构建之间没有协议语义变更，只有实现修复或发布工程修复。

## 阶段 5：版本切换与发布

**目标：** 原子发布可复现的 `v1.0.0`。

1. 全仓切换 `meta.protocolVersion`、Schema 描述和 Renderer 支持版本到 `"1.0"`；
2. 根包、MCP 包、lockfile 和 Docker tag 切到 `1.0.0`；
3. 发布 `0.2` / `0.3` 到 `1.0` 的迁移说明，明确 query 复合值被拒绝、保留参数和 legacy adapter；
4. 执行 `release:check`，清空 CHANGELOG 待发布区并写入 tag、commit SHA、fixture digest；
5. 推送 `v1.0.0`，从远端重新拉取 `1.0.0` 与 SHA 镜像执行 smoke；
6. 记录镜像 digest，不再修改 tag 指向。

## 建议节奏与责任

| 迭代 | 重点 | 主责 | 可并行 |
|---|---|---|---|
| 1 | 阶段 0、G1 ADR、fixture schema | 协议维护者 + 前端 | 后端语言/负责人确认 |
| 2 | G2/G3 ADR、参考 URL 实现、静态拒绝 | 前后端架构组 | G1 fixtures |
| 3 | 请求/搜索/响应 fixtures | 一致性套件维护者 | 前端/后端消费者接入 |
| 4 | reactions/actions/errors/uploads/六场景 | 各能力 owner | 发布检查脚本 |
| 5 | RC 全链验证、迁移文档、版本切换 | 发布负责人 | 干净环境验证 |

每个迭代都必须保持阶段 0 基线为绿。预计 5 个短迭代；若生产后端消费者尚无负责人，G4 和 `v1.0.0` 日期不得承诺。

## 发布决策规则

- **Go：** G1-G4 全关；根/MCP/Docker/前端/后端全部绿；迁移说明和远端镜像 smoke 完成。
- **No-Go：** 任一 fixture 在两个实现间结果不同；存在未知版本 fallback；query 仍允许未标准化复合值；后端只使用复制后的 fixtures；发布 tag 无法由 commit SHA 重建。
