# 审计 0039 修复计划

## 目标

在不扩大未决协议能力的前提下，收敛表达式正文与 L2/L3a/L4 的行为差异，明确 `$parentRow` 与 `$context` 的可实现边界，补齐 Renderer 初始化接口，并将 MCP 20KB 限制落实到最终工具响应。

## 决策优先项

### D1：`$parentRow` 与嵌套表格

必须在实现前选择一条路径：

1. **保留能力：** 为 table 定义唯一嵌套 Node 挂载点，完整同步 DSL、L2、L3a、Renderer 和场景；
2. **暂缓能力：** v0.2 从公开可用变量中移除 `$parentRow`，L3a 明确拒绝，待未来 ADR 定义嵌套结构后再恢复。

验收标准不是某一层脚本通过，而是存在明确的完整页面正例或反例，且 L0-L4 结论一致。

**决策结果：** 选择暂缓能力。v0.2 保守拒绝 `$parentRow.*`；未来需先用新 ADR 定义唯一嵌套挂载结构、父行生命周期和全链路正反例。

### D2：`$context` 生命周期

必须在修改 ADR 前选择一条路径：

1. **静态实例快照：** context 仅在初始化时注入，更新通过重挂载创建新实例；
2. **显式实例内刷新：** Renderer 提供明确更新入口，刷新触发新 Snapshot/Evaluate/Commit 轮次。

最终只能保留一个权威口径，并同步 ADR-0003、ADR-0006、表达式规范与 Renderer 规范。

**决策结果：** 选择静态实例快照。context 更新通过销毁并重挂载 Renderer 生效；不在未定义 `updateContext` API 的情况下引入隐式响应式刷新。

## 实施阶段

1. **阶段一：决策收敛。** 完成 D1/D2，先修改权威文档和 ADR，再调整实现。
2. **阶段二：表达式与上下文校验。** 处理 V117、V118、V121，并建立表单字段类型、Node 树上下文和比较运算符边界的回归样例。
3. **阶段三：结构感知 lint。** 处理 V119，使 L4 区分协议字段与业务字典键，同时保留对字典值对象的约束。
4. **阶段四：Renderer 接口。** 处理 V123，使用单一类型组合避免正文和附录继续双写漂移。
5. **阶段五：MCP 返回预算。** 处理 V124，在最终工具文本边界测试 20KB、UTF-8 与章节列表行为。
6. **阶段六：全链路验证。** 运行根校验正反例、官方场景、MCP build/test/smoke；按变更范围决定是否执行 Docker smoke。

## 测试策略

| 层级 | 最小覆盖 |
|---|---|
| L2 | 表单内/外 visibleWhen dependencies；最终嵌套表格决策的结构正反例 |
| L3a | `$self.start/end`、未知 `$self.*`、比较链、`$parentRow` 最终边界 |
| L4 | tagMap 禁用词同名业务键正例、映射项内部样式字段反例 |
| 统一入口 | 每个协议能力至少一份完整页面；确认 L0-L4 结论一致 |
| MCP | validate_content 对新增边界的映射；get_doc 最终工具文本字节预算 |
| Docker | 仅在 MCP 运行文件、打包内容或镜像文档发生相关变化时执行 |

## 完成条件

1. V117-V124 的 checklist 子项全部标记 `[x]` 并记录实际验证命令；

3. 新增正反例能证明问题已修复，而不只是现有测试继续通过；

5. 用户确认修复结果后才执行归档。
