# 审计报告 0046：`02-reaction-expression.md` 小节编号重复

## 背景

本轮基于 0045 已归档后的再次全量复审。0045 中的 L3a `$self` 漏检、`scope: row` 挂载限制、MCP 官方场景回归与文档卫生项已确认落地，本轮不重复报告 V175-V190。

本轮新问题不是协议语义或实现缺失，而是 0045 回填 `ROW_SCOPE_MOUNT` 章节时引入的文档编号漂移：`docs/02-reaction-expression.md` 的第 10 节下出现两个 `10.6` 小节，导致基于手写编号的引用不再唯一。

扫描活跃区与归档区后，历史最大审计编号为 `0045`，本轮使用 `0046`。全局最大问题编号为 `V190`，因此本轮从 `V191` 开始。

## 发现的问题

### V191（🟢）`02-reaction-expression.md` 出现重复的 `10.6` 小节编号

**位置：** `docs/02-reaction-expression.md` §10.6（`scope: row` 挂载点限制）与紧随其后的 `data.params / optionsSource.params` 小节。

**当前值：** 当前第 10 节下同时存在两个三级标题：

- `### 10.6 scope: row 挂载点限制（ROW_SCOPE_MOUNT）`
- `### 10.6 $deps 出现在非表单 data.params / optionsSource.params 中`

**问题：** 同一层级的小节编号重复，导致正文、CHANGELOG、审计报告或 ADR 若按“§10.6”引用时，无法唯一指向目标内容；Markdown 自动生成的锚点也会因第二个同名编号产生后缀化路径，进一步增加引用歧义。

**影响：** 🟢 轻微。协议语义与脚本实现未受影响，但文档可定位性下降，且该文件是高频被交叉引用的权威规范，后续容易再次诱发锚点漂移类问题。

**建议修复：** 将第二个 `10.6` 顺延为 `10.7`，并同步检查仓库内提到“§10.6”的正文/审计描述，避免把 `ROW_SCOPE_MOUNT` 与 params 规则混写为同一编号。

**与已归档审计的关系：**
- 0026/V26、0041/V141 处理过“章节重排后引用漂移”与“文件存在但章节锚点失效”的同类问题；
- 0045/V190 新增 `ROW_SCOPE_MOUNT` 小节后，本轮确认修复内容已落地，但新增小节编号未顺延，属于**新的编号卫生缺陷**，不是旧问题回退。

**需要同步的文件：** `docs/02-reaction-expression.md`；必要时抽查 `docs/CHANGELOG.md`、`docs/audit/archived/README.md` 及后续引用正文中的“§10.6”表述。
