# 审计报告 0020：组件注册表文档盲区 + 表达式引用漂移

## 背景

本轮审计基于 0019 已归档后的再次全量文档复审。0019 中修复的 V22（ADR-0003 权限表达式旧措辞残留）和 V23（`protocolVersion` 格式校验缺口）在当前工作区已确认修复，本轮不重复报告。

## 发现的问题

### V24（🟡）`upload` 组件文档缺失 `placeholder` 字段定义

**位置：** `docs/03-component-registry.md` `upload` 组件 props 表格；`schemas/component-registry.json` `upload.props`

**当前值：** `03-component-registry.md` 中 `upload` 组件的 props 表格列出了 `field`、`label`/`labelKey`、`accept`、`maxSize`、`multiple`、`action`、`description`、`tooltip`、`required`、`defaultVisible`，**没有 `placeholder`**。

`schemas/component-registry.json` 中 `upload.props` 包含：

```json
"placeholder": { "type": "string", "required": false, "since": "0.2" }
```

**问题：** `placeholder` 字段已在 `component-registry.json` 中定义（`since: "0.2"`），`input`、`inputNumber`、`select`、`datePicker`、`dateRangePicker` 的 doc 表格均有 `placeholder` 行，唯独 `upload` 遗漏。

**影响：** 🟡 中等。配置编写者参考 `03-component-registry.md` 为 `upload` 配置 `placeholder` 时，会因 doc 中找不到该字段而误以为 `upload` 不支持占位提示，迫使改用其他组件或放弃该能力。

**建议修复：** 在 `upload` 组件的 props 表格中补充 `placeholder` 行，标注 `(since 0.2)`。

---

### V25（🟢）`datePicker` / `dateRangePicker` 的 `placeholder` 缺少 `(since 0.2)` 标注

**位置：** `docs/03-component-registry.md` `datePicker` 和 `dateRangePicker` 组件的 props 表格

**当前值：** 两个组件的 doc 表格中 `placeholder` 行均为：

```
| `placeholder` | string | 否 | 占位提示文案 |
```

未标注 `(since 0.2)`。而同一表格中的 `description` 和 `tooltip` 均标注了 `(since 0.2)`。`input`、`inputNumber`、`select` 三个组件的 `placeholder` 也均标注了 `(since 0.2)`。

**问题：** 上下文不一致。`placeholder` 与 `description`、`tooltip` 均为 v0.2.0（B5）引入的可选字段，但仅有 `datePicker`/`dateRangePicker` 的 `placeholder` 遗漏了 `(since 0.2)` 标注。

**影响：** 🟢 轻微。不影响协议理解，但与同文档其他组件的标注风格不一致，且与 `component-registry.json` 中 `since: "0.2"` 的声明不对应。

**建议修复：** 在 `datePicker` 和 `dateRangePicker` 的 `placeholder` 行补充 `(since 0.2)`。

---

### V26（🟢）`02-reaction-expression.md` §9.1 交叉引用漂移

**位置：** `docs/02-reaction-expression.md` §9.1 作用域隔离规则

**当前值：**

```md
- **`$row` 与 `$deps` 互斥**：`scope: row` 的表达式中不能出现 `$deps.*`；
  `scope: form` 的表达式中不能出现 `$row.*`。违反该规则的配置由静态校验直接拒绝
  （见 §10.1）。
```

**问题：** §10.1 的标题是 `$deps` 出现在非表单 `visibleWhen` 中，仅覆盖"非表单节点 `visibleWhen`"中 `$deps` 的静态拒绝规则，**不覆盖** `scope: row` 与 `scope: form` 互斥这一更通用的场景。§9.1 所描述的是作用于 `reactions`、`visibleWhen`、`permissions` 等多个位置的通用互斥规则，引用到仅覆盖单个位置的 §10.1 属交叉引用范围错误。

**影响：** 🟢 轻微。规则正文本身明确无误（"违反该规则的配置由静态校验直接拒绝"），附录 A 变量可见性矩阵也已完整覆盖。引用不精确仅影响读者快速定位到对应的静态校验规则条目。

**建议修复：** 将 `（见 §10.1）` 改为 `（见 §10 静态校验规则）` 或直接删除该引用——§9.1 正文已足够明确，无需额外交叉引用。

### V27（🟢）`01-node-protocol.md` §3.3 `responseMapping` 章节引用精度

**位置：** `docs/01-node-protocol.md` §3.3 `data` 字段的 `responseMapping` 说明注释

**当前值：**

```yaml
responseMapping: map # 【可选，since 0.2.4】响应字段名映射，见 04-datasource-contract.md §4.1
```

**问题：** `04-datasource-contract.md` 中 `responseMapping` 的具体规范说明在 **§4.1.1**（响应字段名映射），而非 **§4.1**（列表类接口）。§4.1 是"列表类接口"章节，不直接包含 `responseMapping` 的定义。

**影响：** 🟢 轻微。读者按 `§4.1` 跳转后仍可在同节内找到 `responseMapping` 内容（§4.1.1 紧随 §4.1），且 Markdown 锚点链接可以精确跳转。但章节号不够精确，与文档自身追求精确引用的风格不符。

**建议修复：** 将 `§4.1` 改为 `§4.1.1`。

## 防止复发的建议

1. **组件字段一致性自检：** 新增组件或为已有组件新增 props 字段时，应同时核对 `03-component-registry.md`（字段表格）和 `component-registry.json`（字段定义），避免任一方遗漏。
2. **`since` 标注交叉检查：** 字段表格中的 `(since X.Y)` 应与 `component-registry.json` 中对应字段的 `since` 值一致，可通过 CI 脚本生成差异报告。
3. **交叉引用精度复核：** 使用 "见 §" 或 "见 " 模式搜索全文，每次章节重排后需同步更新所有交叉引用。
