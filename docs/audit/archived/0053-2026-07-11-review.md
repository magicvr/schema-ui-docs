# 审计报告 0053：`$context` 可用位置 + `contains` 静态规则残留 + dependencies Schema 描述 + 文档卫生

## 背景

本轮基于 0052 已归档后的再次全量复审。0052 中的 ADR-0003 `contains` 函数式语法残留（V206）已确认落地；V197–V205 相关修复（params 整值替换、`$self` 禁令、`contains` L3a 右操作数、L3a 表格字段异常分类等）仍在位；六个官方场景 YAML 通过 L0–L4；MCP 实现与 `06` / ADR-0007 无明显漂移。

本轮新问题集中在：

1. 表达式入口变量表将 `$context.*` 标为「所有位置」，与 `params` 值替换禁令冲突；
2. V205 执行层已落地，但权威静态规则章（`02` §10）与 Renderer 校验清单（`08` §5.5）未收录；
3. V164 后 `reaction.schema.json` 的 `dependencies` 描述仍未写明「无 `$deps.` / `$row.` 前缀」；
4. MCP CHANGELOG 路径、`$parentRow`「暂不支持」措辞、`02` §1 链到错误章节等卫生项。

扫描活跃区与归档区后，历史最大审计编号为 `0052`，本轮使用 `0053`。扫描所有 `*-review.md`、`*-checklist.md` 与已有 `*-plan.md` 后，全局最大问题编号为 `V206`，因此本轮从 `V207` 连续编号到 `V212`。

## 发现的问题

### V207（🟡）`$context`「所有位置」与 `params` 禁令矛盾

**位置：**

- `docs/02-reaction-expression.md` §2 变量表（`$context.user.*` / `$context.features.*` 的「可用位置」列）
- 对照：同文档附录 A、§10.7；`08-renderer-spec.md` §5.5

**当前值：**

- §2：`$context.user.*` / `$context.features.*` 的可用位置写为 **「所有位置」**
- 附录 A / §10.7：`data.params` / `optionsSource.params` 中 **`$context.*` 为 ❌**，仅允许字面量或完整单个 `$deps.*` 整值替换

**问题：** §1 已把 `data.params` / `optionsSource.params` 列为表达式相关使用位置。入口变量表写「所有位置」，与后文静态规则和矩阵直接冲突。实现者或生成配置若只读 §2，可能在 params 中写入 `$context.user.id` 一类值，随后被 L3a 拒绝。

**影响：** 🟡 中等。不破坏现有 L3a 行为，但入口权威表误导配置生成与跨实现阅读。

**建议修复：**

1. 将 `$context.*` 的「可用位置」改为条件表达式挂载点（`reactions` / `visibleWhen` / `permissions` / 表格列与操作表达式等），明确 **不含** `params` 值替换。
2. 可交叉引用附录 A / §10.7。

**需要同步的文件：**

- `docs/02-reaction-expression.md`
- `docs/CHANGELOG.md`

**与已归档审计的关系：** V65/V197 闭合了 params 变量与整值替换规则，但未回改 §2 变量表中「所有位置」的笼统表述。

### V208（🟡）V205 后 `contains` 右操作数未写入 `02` §10 / `08` §5.5

**位置：**

- `docs/02-reaction-expression.md` §3、§10
- `docs/08-renderer-spec.md` §5.5
- 对照：`docs/06-validation.md` L3a 行与自检清单；`scripts/validate-l3a-expressions.js`

**当前值：**

- §3 写「右操作数字面量」，未枚举允许类型，也未列为 §10 独立静态规则
- §10 / `08` §5.5 的「至少包括」清单 **没有** `contains` 右操作数约束
- `06` + L3a 已明确：右操作数仅字符串 / 数字 / 布尔 / `null`，拒绝变量与分组表达式

**问题：** V205 同步了实现与校验文档，但表达式权威静态规则章与 Renderer 加载时校验清单仍缺该项。只读 `02`/`08` 的 Renderer 作者可能漏实现，与 L3a/CI 行为分叉。

**影响：** 🟡 中等。协议权威章节与工具链不对齐，跨实现互操作性风险。

**建议修复：**

1. 在 `02` 新增 **§10.8**（或等价条目）：右操作数必须是 §4 所列字面量类型；拒绝变量与分组表达式。
2. 在 `08` §5.5 补一条与 `06` 同口径的 bullet。
3. §3 正文补「字符串 / 数字 / 布尔 / `null`」枚举，与 L3a 错误文案一致。

**需要同步的文件：**

- `docs/02-reaction-expression.md`
- `docs/08-renderer-spec.md`
- `docs/CHANGELOG.md`

**与已归档审计的关系：** 0051/V205 执行了 L3a 与 `06`；本项是权威正文 / Renderer 清单的文档层残留。

### V209（🟡）`reaction.schema.json` 的 `dependencies` 描述未写明「无前缀」路径

**位置：**

- `docs/schemas/reaction.schema.json` `properties.dependencies.description`
- 对照：`docs/02-reaction-expression.md` §8.1；`docs/01-node-protocol.md` §3.5；`docs/06-validation.md` L3a

**当前值：**

> scope: form（默认）下列出 when 表达式中使用的 $deps.* 表单字段；scope: row 下列出 when 表达式中使用的 $row.* 行内字段路径……

**问题：** V164 已在正文明确：`dependencies` 写 **前缀之后** 的路径（如 `orderType` / `canRefund` / `user.id`），**不得**写 `"$deps.orderType"` 或 `"$row.canRefund"`。机器可读 Schema 描述仍沿用 V15 的笼统措辞，易被读成「数组内写带前缀的变量」。

**影响：** 🟡 中等。L3a 会拒绝带前缀的错误声明，但 Schema 作为机器可读权威描述会误导生成器与阅读者。

**建议修复：** 更新 `dependencies.description`，与 §8.1 对齐：form 写字段名（无 `$deps.` 前缀）；row 写 `$row.` 之后完整点路径（无 `$row.` 前缀）；未引用 `$deps.*` / `$row.*` 时可为空数组，例如仅引用 `$self`、`$context.*` 或不引用变量。

**需要同步的文件：**

- `docs/schemas/reaction.schema.json`
- `docs/CHANGELOG.md`

**与已归档审计的关系：** 0016/V15 首次写明 form/row 语义；0044/V164 只更新了 `01`/`02`/`06`，未回写 Schema 描述。

### V211（🟢）组件表仍写「暂不支持 `$parentRow.*`」

**位置：**

- `docs/03-component-registry.md` `RowAction.requestMapping`
- `docs/schemas/component-registry.json` 对应 description
- 对照：`02` 禁止事项、`06` L3a、CHANGELOG Unreleased 说明、ADR-0004 D5

**当前值：** 「v0.2 **暂不支持** `$parentRow.*`」

**问题：** 权威口径为 v0.2 **全面 / 静态拒绝** `$parentRow.*`（无嵌套表格挂载结构）。「暂不支持」像软缺能力，易被理解为可写但运行时忽略。

**影响：** 🟢 轻微术语漂移。

**建议修复：** 统一为「v0.2 静态拒绝 `$parentRow.*`（无嵌套表格挂载结构）」。

**需要同步的文件：**

- `docs/03-component-registry.md`
- `docs/schemas/component-registry.json`
- `docs/CHANGELOG.md`

**与已归档审计的关系：** 0039/V120 统一拒绝策略；组件表/DSL 描述未完全改用语。

### V212（🟢）`02` §1 将「整值替换」链到 `04` §3.2 而非 §3.1

**位置：** `docs/02-reaction-expression.md` §1 表「数据请求参数」行

**当前值：** 摘要写「仅整值替换，禁止模板拼接」，链接为 `04-datasource-contract.md` **§3.2**（作用域边界）

**问题：** 整值替换 / 模板禁令 / 空值省略主规则在 **§3.1**；§3.2 是表单/非表单作用域。锚点有效但语义错位。

**影响：** 🟢 轻微。读者点进去先看到作用域，找不到整值替换主规则。

**建议修复：** 主链改为 §3.1（可保留对 §3.2 的次要说明，或一句并链）。

**需要同步的文件：**

- `docs/02-reaction-expression.md`
