## V23（🟡）`meta.protocolVersion` 的 MAJOR.MINOR 格式要求未落入 `page.schema.json`

| 字段 | 值 |
|---|---|
| **位置** | `01-node-protocol.md` §2；`00-overview.md` §5；`08-renderer-spec.md` §3.2；`schemas/page.schema.json` `meta.protocolVersion` |
| **问题** | 正文要求 `protocolVersion` 为 MAJOR.MINOR 且不含 PATCH，但 `page.schema.json` 只校验字符串，无法拒绝 `"0.2.4"`、`"v0.2"`、`"foo"` 等非法格式 |
| **操作** | 在 `page.schema.json` 中为 `meta.protocolVersion` 增加格式约束；必要时同步 `06-validation.md` 对 L0 能力边界的说明 |
| **文件** | `docs/schemas/page.schema.json`，必要时补充 `docs/06-validation.md` |

- [x] V23a 为 `page.schema.json` 的 `meta.protocolVersion` 增加 MAJOR.MINOR 格式约束。修改：新增 `pattern: "^[0-9]+\\.[0-9]+$"`。
- [x] V23b 确认约束职责只覆盖格式，不把具体支持版本范围硬编码到 Schema（除非决定改用 `enum`）。结果：未新增 `enum`，版本支持范围仍由 Renderer 协商处理。
- [x] V23c 核对 `06-validation.md` 的 L0 页面结构校验说明是否需要补充 `protocolVersion` 格式校验。修改：L0 校验内容补充 `meta.protocolVersion` MAJOR.MINOR 格式。
- [x] V23d 用一个合法值和至少一个非法值验证 `page.schema.json` 的校验行为。结果：`0.2` 通过，`0.2.4` 被 `pattern` 拒绝。
