# 审计报告 0052：ADR-0003 `contains` 函数式语法残留

## 背景

本轮基于 0051 已归档后的再次全量复审。0051 中的 L3a 表格字段异常分类（V204）与 `contains` 右操作数字面量约束（V205）已确认落地；MCP 115 项测试、TypeScript 构建与 tools smoke 均通过。

本轮发现 Accepted ADR-0003 的 `$context` 属性链容错说明仍将 `contains` 描述为“方法调用”，并使用 `contains(...)` 形式。当前协议正文、Renderer 规范与 L3a 已统一规定 `contains` 只能作为中缀二元比较运算符使用，并禁止任何函数调用，因此该 ADR 残留会误导实现者重新开放非标准语法。

扫描活跃区与归档区后，历史最大审计编号为 `0051`，本轮使用 `0052`。扫描所有 `*-review.md`、`*-checklist.md` 与已有 `*-plan.md` 后，全局最大问题编号为 `V205`，因此本轮从 `V206` 开始编号。

## 发现的问题

### V206（🟡）Accepted ADR 将中缀 `contains` 描述为方法调用

**位置：**

- `docs/decisions/0003-context-namespace-and-visible-when.md` D1a“属性链容错（补充）”
- 对照 `docs/02-reaction-expression.md` §2、§3
- 对照 `docs/08-renderer-spec.md` §5.2、§5.4

**当前值：**

ADR-0003 当前写道：

> 若访问路径中任意一环为 `undefined`，后续的属性访问和方法调用（如 `.roles`、`contains(...)`）一律短路返回 `undefined`，不抛出运行时异常。

当前正式协议则明确：

- `contains` 是中缀二元比较运算符，语法为 `a contains b`；
- `contains(a, b)` 不是合法协议语法；
- 表达式禁止所有函数调用。

**问题：** Accepted ADR 把属性访问 `.roles` 和不存在的函数式 `contains(...)` 并列为“方法调用”，既错误描述了属性链传播，也展示了协议明确禁止的调用形式。实现者若以 ADR 作为设计依据，可能把 `contains(...)` 当成允许的特殊函数语法。

**影响：** 🟡 中等。当前协议正文、Renderer 示例和 L3a 行为是正确的，不会直接造成现有校验绕过；但 ADR 是已接受的设计依据，这处残留会导致不同 Renderer 重新产生表达式语法分歧，并削弱“禁止函数调用”的安全边界。

**建议修复：**

1. 将 D1a 改为仅描述属性路径的可选链式传播，不再使用“方法调用”措辞或 `contains(...)` 形式。
2. 明确路径访问得到 `undefined` 时，作为 `contains` 等比较运算符的操作数，比较结果按 D1a 主规则转为 `false`，且不抛异常。
3. 全仓搜索 `contains(`，确认剩余命中只用于明确说明或测试非法函数语法，不再有规范性文字将其当作合法调用。
4. 在 `docs/CHANGELOG.md` Unreleased 记录 ADR 同步修复，并同步 ADR `date` 元数据。

**需要同步的文件：**

- `docs/decisions/0003-context-namespace-and-visible-when.md`
- `docs/CHANGELOG.md`

**与已归档审计的关系：** 0016/V16 修复的是 `08-renderer-spec.md` 将 `contains` 注册为函数并使用“除 contains 外禁止函数调用”的措辞；0039/V121 修复的是比较运算符链式使用；0051/V205 执行的是 `contains` 右操作数字面量约束。三轮均未覆盖 ADR-0003 D1a 的函数式残留，因此本项不是重复报告。
