# 审计报告 0035：`source: ref` 继承 responseMapping 校验缺口 + v0.2.8 版本漂移 + Renderer permissions 白名单描述落后

## 发现的问题

### V88（🟡）`source: ref` 未解析继承的 `datasources.*.responseMapping`，列表/分页语义校验仍可被绕过

**位置：**
- `scripts/validate-l2-components.js` `validateResponseMapping()`：仅读取 `node.data.responseMapping`，不解析 `doc.datasources[data.ref]`
- 对照文档：
  - `docs/01-node-protocol.md` §3.3：`source: ref` 可引用页面级 `datasources`，并可在引用目标为 API 时携带本地 `responseMapping` 覆盖响应解析
  - `docs/04-datasource-contract.md` §1 / §4.1.1：`datasources` 为预声明 `DataRef`；`responseMapping` 用于列表/分页字段映射
  - `docs/06-validation.md` §1：L2 负责 `source: api` / `source: ref` 本地 `responseMapping` 条件必填
- 对照实现：V71 已把条件必填扩展到“节点本地声明了 `responseMapping` 的 `api` 或 `ref`”，但**不解析**引用目标上的映射

**当前值（实测）：**

1. 页面级：
   ```yaml
   datasources:
     orders:
       source: api
       url: /api/orders
       responseMapping:
         total: result.total   # 缺 list
   body:
     type: table
     props:
       pagination: { mode: server, pageSize: 10 }
       ...
     data:
       source: ref
       ref: orders             # 无本地 responseMapping
   ```
   → L0/L1、L2、L3a、L4 全链路 `passed`

2. 同类配置将 `table` 换成 `chart`，`datasources` 仅声明 `responseMapping.total`、无 `list` → 同样全链路通过

3. 若节点本地再写 `responseMapping: { total: other.total }`（无 `list`），L2 会正确拒绝——说明规则只覆盖“本地声明”，不覆盖“继承声明”

**问题：** Renderer 在 `source: ref` 时会使用 `datasources[ref]` 上的请求与响应解析配置；列表/分页语义条件因此同样应作用于**生效的** `responseMapping`（本地覆盖优先，否则继承预声明）。当前 L2 只在节点自身出现 `responseMapping` 时检查，导致把不完整映射上移到 `datasources` 即可绕过 V71。

**影响：** 🟡 中等。不会破坏 Schema 结构，但会让服务端分页表格/图表在 CI 与 MCP `validate_content` 中误报通过，运行时才因缺少 `list`/`total` 映射进入错误态。与 V71 同属 `responseMapping` 语义校验缺口的残留面。

**建议修复：**
1. L2 在 `data.source: ref` 时解析生效映射：
   - 若节点本地声明 `responseMapping`，继续使用本地值（覆盖语义）；
   - 否则若 `doc.datasources[data.ref]` 存在且声明了 `responseMapping`，使用该预声明映射；
   - 仅当生效映射存在时，对 `table`/`chart` 执行 `list` 条件必填，对 `table.props.pagination.mode: server` 执行 `total` 条件必填。
2. 同步 `docs/06-validation.md` L2 / v0.2.4 或后续变更说明：明确“本地或继承的 `responseMapping` 均进入条件必填”。
3. 补充 MCP `validate_content` 反例：
   - `table + source: ref + datasources.*.responseMapping.total only` 应失败；
   - `chart + source: ref + datasources.*.responseMapping.total only` 应失败；
   - 本地合法覆盖 / 预声明合法 `list`+`total` 应通过。

**需要同步的文件：**
- `scripts/validate-l2-components.js`
- `docs/06-validation.md`
- `mcp/tests/validate-content.test.ts`
- `mcp/tests/test-utils.ts`

**需要同步的文件：**

### V90（🟢）`08-renderer-spec.md` §5.5 对 `permissions.*` 仍写“只允许 `$context.*`”，未同步到 `user`/`features` 根白名单

**位置：**
- `docs/08-renderer-spec.md` §5.5：
  > `permissions.*` 表达式中不能出现 `$deps.*`，只允许 `$context.*`。
- 对照：
  - `docs/06-validation.md` L3a：已写 `permissions.*` 仅使用 `$context.user.*` / `$context.features.*`
  - `scripts/validate-l3a-expressions.js`：`PERM_CONTEXT_ONLY` + `UNKNOWN_CONTEXT_NAMESPACE`（V73/V74）
  - `docs/02-reaction-expression.md` §2 / §12：`$context` 根命名空间白名单

**当前值：** Renderer 规范清单仍停留在“禁止 `$deps.*`、允许任意 `$context.*`”的旧口径。

**问题：** V73/V74 已把权限表达式收紧为仅 `$context.user.*` / `$context.features.*`。`08` 是 Renderer 实现的操作清单，若不同步，接入方可能按旧描述放过 `$context.tenant.*` 等路径，与 CI/MCP 行为不一致。

**影响：** 🟢 轻微。实际 L3a 脚本已正确 enforce；问题主要是 Renderer 规范文档落后，增加实现误读风险。

**建议修复：**
1. 将 `08-renderer-spec.md` §5.5 中 permissions 条目改为：
   - `permissions.*` 只允许 `$context.user.*` / `$context.features.*`；
   - 禁止 `$deps.*`、`$self`、`$row.*`、`$parentRow.*` 及未登记的 `$context` 根命名空间。
2. 同步 `last_updated`（若本轮修改该文件）。

**需要同步的文件：**
- `docs/08-renderer-spec.md`

## 与已归档审计的关系

- V88 与 V71/V86 相关：V71 补了节点本地 `source: ref` 映射条件必填；V86 补了 `data.ref` 存在性；本轮确认“映射写在 `datasources`、节点只 ref”时条件必填仍可被绕过。
- V89 与 V1/V39/V42/V87 同类：正文或校验说明先写入新 patch/元数据，入口版本声明与发布物未同步。0034 在 `06` 落了 v0.2.8 标签，但未配套 CHANGELOG/总纲/MCP 版本。
- V90 与 V68/V73/V74 相关：V68 曾整体刷新 Renderer L3 清单，V73/V74 进一步收紧 permissions；`08` §5.5 的 permissions 行未跟到最终白名单。

## 建议处理顺序

1. **先修 V88**：补齐 L2 对继承 `responseMapping` 的解析与条件必填，并加 MCP 反例。
2. **再修 V89**：以最终合入的校验能力为准写 `v0.2.8` CHANGELOG，并同步 `00-overview` 与 MCP 包/文档版本。
3. **最后修 V90**：更新 `08` §5.5 permissions 描述。

## 防止复发的建议

1. `responseMapping` 语义校验必须以“生效映射”为准：本地覆盖优先，否则继承 `datasources[ref]`；单测需同时覆盖本地声明与预声明继承两条路径。
2. 凡在协议正文写入 `v0.2.x 变更` 标签，同一审计 checklist 必须包含：`CHANGELOG` 条目、`00-overview` 最新补丁声明、以及是否 bump MCP 包版本的明确决策。
3. `08-renderer-spec.md` §5.5 应视为 L3a 规则的同步镜像；L3a 规则变更时 checklist 显式包含 Renderer 清单 diff。
