# 审计 0041 修复计划

## 目标

在不扩大 v0.2 协议能力边界的前提下，闭合 V130-V141：优先修复已由正文明确规定但实现未执行的规则；涉及新增公共字段或返回结构时，先记录单一决策，再同步正文、机器契约、实现和测试。

## 决策点

### D1：普通表单 GET request

**建议选择：** v0.2 禁止 `form.props.submitAction` 引用 `method: GET` 的 request。

理由：当前普通 Action 只定义 JSON `bodyMapping`，而行级 Action 已有独立 query 映射。此时为普通表单补 `queryMapping` 会扩大 Action 公共结构和 Renderer API；保守拒绝能以最小变更恢复可执行性，未来真实场景出现后再通过 ADR 扩展。

### D2：日期边界校验强度

**建议选择：** 同时验证严格 `YYYY-MM-DD` 语法和真实日历日期。

理由：只做正则会放行 `2026-99-99`，仍不能兑现“ISO 日期”语义。实现可采用显式年月日解析和 UTC 往返校验，避免宿主本地时区差异。

### D3：validate_content 返回预算

**建议选择：** 保持现有 `layers` 结构，每层按顺序保留可容纳的违规项，并新增顶层 `truncated` 与每层统计；最终以工具 JSON 文本 UTF-8 字节数为准。

理由：结构化裁剪能保持 JSON 可解析和现有消费者基本兼容；按层保留可避免 L0/L1 大量错误吞掉全部 L2-L4 证据。
