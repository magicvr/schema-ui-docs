## A. 直接修复（bug / 文档矛盾）→ v0.2.0

- [x] **A1. JSON Schema CSS 禁用词校验修复（双轨策略）**（审计 §2.2）
  - `node.schema.json` 中 `"not": {"required": [...]}` 语义错误，只禁止多个同时出现
  - **决策：** 保留并修复 Schema 层（`not` + `anyOf` 逐一拦截）+ L4 lint 脚本做深度补充
  - 涉及文件：`schemas/node.schema.json`、`06-validation.md`（明确 L1/L4 双轨防线关系）
- [x] **A2. 修复 `text` 组件 `data` 支持说明矛盾**（审计 §4.3）
  - `supportsData: false` 与注释矛盾 → 改为 `supportsData: true`
  - 涉及文件：`03-component-registry.md`、`schemas/component-registry.json`
- [x] **A3. `valueField` 从 `DataRef` 移至组件 `props`**（审计 §2.1）
  - ⚠️ **破坏性变更**，需附带迁移脚本
  - 涉及文件：`01-node-protocol.md`、`03-component-registry.md`、`schemas/node.schema.json`、`schemas/component-registry.json`、`05-scenarios/grid-dashboard.md`
- [x] **A4. 补充 `datasources` + `data.ref` 使用示例**（审计 §4.1）
  - 涉及文件：`05-scenarios/grid-dashboard.md`
- [x] **A5. `meta` 增加 `protocolVersion` 字段**（审计 §4.2）
  - 必填 `protocolVersion: string`（如 `"0.2"`），Renderer 版本兼容锚点
  - 涉及文件：`01-node-protocol.md`、`schemas/node.schema.json`、所有场景示例

## B. 直接新增（向后兼容，不扩展表达式边界）→ v0.2.0

- [x] **B1. Node 增加可选 `id` 字段**（审计 §3.1）
  - `id: string`，页面内唯一，用于定位/埋点/增量 patch
  - 涉及文件：`01-node-protocol.md`、`schemas/node.schema.json`
- [x] **B2. `tabs` children 归属 → 方案 A：`items[].content` 内嵌**（审计 §2.4）
  - 不再依赖顶层 `children` 做归属匹配，不为 tabs 单独引入通用 `key`
  - 涉及文件：`03-component-registry.md`、`schemas/component-registry.json`
- [x] **B3. `actions` 完整契约补充**（审计 §2.5）
  - type 枚举：`request` / `navigate` / `modal` / `custom`（custom 仅白名单预注册）
  - `onSuccess`/`onError` 行为：`toast` / `navigate` / `reload` / `closeModal`
  - 涉及文件：新增 `07-actions-contract.md`、`schemas/action.schema.json`、`05-scenarios/form-with-reactions.md`
- [x] **B4. `span` 提升为通用 props**（审计 §3.7）
  - 任何 `grid` 直接子节点均可在 `props` 中声明 `span`
  - 涉及文件：`03-component-registry.md`
- [x] **B5. 表单字段补充 `placeholder` / `description` / `tooltip`**（审计 §4.4）
  - 原 P3，因改动成本低并入本批
  - 涉及文件：`03-component-registry.md`、`schemas/component-registry.json`
- [x] **B6. i18n `labelKey` 支持**（审计 §3.3）
  - `label`/`title`/`content` 等均可用 `xxxKey` 替代，Renderer 优先查 i18n 词典
  - 涉及文件：`01-node-protocol.md`、`03-component-registry.md`
- [x] **B7. `select` 远程动态选项 `optionsSource`**（审计 §3.4）
  - 新增 `optionsSource`（url/params/labelField/valueField/searchable），`params` 复用 `$deps.*`
  - **空值规则：** `$deps.*` 为 null/undefined 时该参数整体省略
  - 涉及文件：`03-component-registry.md`、`04-datasource-contract.md`
- [x] **B8. 组件级 `states`（空态/加载态/错误态）**（审计 §3.5）
  - Node 新增 `states: { loading, empty, error }`，仅允许语义级字段
  - **范围约束：** 仅 `supportsData: true` 的组件有意义，注册表增加 `supportsStates` 标注
  - 涉及文件：`01-node-protocol.md`、`schemas/node.schema.json`、`03-component-registry.md`、`schemas/component-registry.json`
- [x] **B9. `component-registry.json` 增加 `deprecated`/`since` 标注**（审计 §5.4）
  - 原属"长远建议"，现已正式纳入 v0.2
  - 涉及文件：`schemas/component-registry.json`、`03-component-registry.md`
- [x] **B10. 表格行级操作显隐过渡方案 `visibleField`**（审计 §3.6 过渡方案）
  - 后端下发 `canXxx: boolean`，`table.props.actions[].visibleField` 声明依据字段
  - 数据驱动显隐，不引入 `$row` / `reactions` / `when`
  - 涉及文件：`03-component-registry.md`、`05-scenarios/data-table.md`、`schemas/component-registry.json`
