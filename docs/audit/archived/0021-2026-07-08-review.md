## 发现的问题

### V28（🟡）`03-component-registry.md` 表格列/操作"作用域说明"缺少"仅限表单上下文"前提限定

**位置：** `docs/03-component-registry.md` — `table` 组件 `ColumnDef`/`RowAction` 说明之后的"作用域说明（since 0.2.1）"段落

**当前值：**

```
**作用域说明（since 0.2.1）：** 列/操作内的 `visibleWhen`/`reactions`/`permissions` 可通过 `scope`
属性声明求值作用域：
- `scope: form`（默认）：表达式在表单级求值，可访问 `$deps.*`（表单字段），不可访问 `$row.*`。
- `scope: row`（显式声明）：……
```

**问题：** 上述说明直接声明"默认 `scope: form` 可以访问 `$deps.*`（表单字段）"，未附加任何前提条件。然而 `02-reaction-expression.md` §9.1 和 `04-datasource-contract.md` §3.2 均明确规定：`$deps.*` 仅在**表单上下文**（即节点位于 `form` 的 `children` 树中）合法，在独立表格中使用 `scope: form` + `$deps.*` 会被静态校验直接拒绝。

实际上，`03-component-registry.md` 同文件描述了将 `table` 嵌入 `form.children` 的搜索表单场景（`mode: search`），这是 `table` 列中 `scope: form` + `$deps.*` **唯一合法的使用场景**。但"作用域说明"段落与这一限制完全脱节，未作任何交叉说明。

读者若参照此段落，在独立表格（`body` 下的独立 `table`，非 `form` 子节点）的列定义中使用默认 `scope: form` 并声明 `$deps.*`，配置将无法通过静态校验，且错误信息可能不够直观（因为读者有文档依据）。

**影响：** 🟡 中等。易误用场景为：开发者在独立表格中尝试用 `scope: form` + `$deps.*` 实现列的条件显隐，在本地测试或 CI 阶段被静态校验拒绝，需花时间排查根因。此问题可能在表格与表单共存的页面中尤为隐蔽。

**建议修复：** 在"作用域说明"的 `scope: form` 条目中补充前提说明，内容示例：

> `scope: form`（默认）：表达式在表单级求值，可访问 `$deps.*`（表单字段），不可访问 `$row.*`。**注意：仅当表格本身位于 `form.children` 内（如搜索表单嵌入表格场景）时，`$deps.*` 才合法；独立表格的列/操作中即使声明 `scope: form`，`$deps.*` 仍被静态校验拒绝（见 [02-reaction-expression.md §9.1](./02-reaction-expression.md#91-作用域隔离规则)）。**

### V29（🟢）`CHANGELOG.md` v0.2.4 条目缺少"涉及的文档与 Schema"段落

**位置：** `docs/CHANGELOG.md` — `v0.2.4` 版本条目末尾

**当前值：** `v0.2.4` 条目以"修订："小节结尾，无"涉及的文档与 Schema："段落。

**对比：** `v0.1.0`、`v0.2.0`、`v0.2.1`、`v0.2.2`、`v0.2.3` 每个版本条目末尾均有：

```markdown
**涉及的文档与 Schema：**
- 协议文档：…
- JSON Schema：…
```

唯独 `v0.2.4` 缺失该段落。

**问题：** 格式不一致，读者无法通过快速扫描 CHANGELOG 了解 v0.2.4 涉及了哪些文件，降低了版本追踪的可读性。

**影响：** 🟢 轻微。不影响协议理解或配置编写，但降低 CHANGELOG 维护一致性，且 v0.2.4 的文件范围（`01`/`02`/`04`/`08` 文档 + `node.schema.json` + 两个 ADR 文件）较分散，缺少该段落时读者需要逐条阅读"新增/修订"才能拼凑出受影响文件列表。

**建议修复：** 在 `v0.2.4` 条目末尾补充"涉及的文档与 Schema："段落，内容如下：

```markdown
**涉及的文档与 Schema：**
- 协议文档：`01-node-protocol.md`（删除"求值时序未定义"限制，改为引用 ADR-0006）/`02-reaction-expression.md`（新增 §13 求值时序模型）/`04-datasource-contract.md`（`responseMapping` 升为正式契约）/`08-renderer-spec.md`（增加 `responseMapping` 处理与批量提交/循环保护要求）
- JSON Schema：`schemas/node.schema.json`（`DataRef` 增加 `responseMapping` 字段）
- 新增决策：`decisions/0005-response-mapping.md`、`decisions/0006-expression-evaluation-order.md`
```
