# 审计报告 0019：权限表达式 ADR 残留 + protocolVersion 格式校验边界

## 发现的问题

### V22（🟡）ADR-0003 的 `permissions` 说明仍保留“协议不强制禁止”的旧措辞

**位置：** `decisions/0003-context-namespace-and-visible-when.md` D5；`01-node-protocol.md` §3.9；`02-reaction-expression.md` §10.2

**当前值：** ADR-0003 D5 同一节中先声明：

```md
`permissions.*` 的表达式中禁止出现 `$deps.*`，只允许使用 `$context.*`。解析器在静态校验阶段拦截违反该规则的协议，直接拒绝而非运行时警告。
```

但紧接着在定位区别中仍写：

```md
`permissions` 回答"当前用户能不能对这个节点/操作做某件事"（纯粹基于 `$context.user`，不掺杂业务字段状态，虽然协议不强制禁止，但建议实践上只使用 `$context.user` ...）
```

**问题：** “虽然协议不强制禁止”是旧的建议性约束残留，已与同一 ADR 前文、`01-node-protocol.md` 和 `02-reaction-expression.md` 的当前强制规则冲突。当前协议已经明确 `permissions.*` 仅允许 `$context.*`，出现 `$deps.*` 应被静态校验拒绝。

**影响：** 🟡 中等。核心正文和表达式规范已正确，但 ADR 是决策依据文档；同一节内部自相矛盾会让实现方误以为 `permissions.*` 混入业务字段只是“不推荐”，而不是协议格式错误。

**建议修复：** 删除旧的“虽然协议不强制禁止，但建议实践上只使用 `$context.user`”措辞，改为与强制规则一致的描述，例如“仅允许 `$context.*`，不得掺杂业务字段状态”。

### V23（🟡）`meta.protocolVersion` 的 MAJOR.MINOR 格式要求未落入 `page.schema.json`

**位置：** `01-node-protocol.md` §2；`00-overview.md` §5；`08-renderer-spec.md` §3.2；`schemas/page.schema.json` `meta.protocolVersion`

**当前值：** 正文明确要求：

```md
`meta.protocolVersion` ... 格式为 MAJOR.MINOR（如 `"0.2"`），不含 PATCH 号。
```

`00-overview.md` 也说明 v0.2.x 所有补丁版本共享同一 `protocolVersion` 值 `"0.2"`。但 `schemas/page.schema.json` 中当前约束只有：

```json
"protocolVersion": {
  "type": "string",
  "description": "必填（since 0.2），如 \"0.2\"，Renderer 版本兼容锚点"
}
```

**问题：** `page.schema.json` 只能保证 `protocolVersion` 存在且为字符串，不能拒绝 `"0.2.4"`、`"v0.2"`、`"foo"` 等不符合 MAJOR.MINOR 的值。历史审计已覆盖 `protocolVersion` 必填性和示例缺漏，但没有覆盖格式约束。

**影响：** 🟡 中等。Renderer 的版本协商依赖该字段作为解析规则锚点；格式漂移会让 L0 页面结构校验放过错误版本声明，直到运行时才可能暴露，降低 CI 对协议版本边界的保护力。

**建议修复：** 在 `schemas/page.schema.json` 为 `meta.protocolVersion` 增加 MAJOR.MINOR 格式约束，例如 `pattern: "^[0-9]+\\.[0-9]+$"`。如当前仅支持 v0.2，也可进一步使用 `enum: ["0.2"]`，但这会增加未来协议升级时的 Schema 维护成本；更保守的做法是先校验格式，版本支持范围仍由 Renderer 协商处理。

## 防止复发的建议

1. **强制规则升级后清理旧建议措辞：** 凡是“建议”提升为“静态校验拒绝”的规则，应全文搜索“建议”“不强制”“可选”等旧措辞。
2. **正文格式要求同步到 Schema：** 正文中出现“格式为”“不含”“仅允许”等机器可判定规则时，同步检查对应 Schema 是否已有 `pattern`、`enum` 或等价约束。
3. **区分版本格式与支持范围：** `page.schema.json` 负责拒绝格式错误，Renderer 版本协商负责判断具体 MAJOR.MINOR 是否被支持，避免把两个职责混在一起。
