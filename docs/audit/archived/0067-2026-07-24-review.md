# 审计报告 0067：v2.4 recordView 交付原子一致性

## 结论

**机器层（L2 / registry / fixtures / 能力表）对 ADR-0024 主路径已基本落地；权威文档层与发布门禁勾选存在原子不一致，构成发布阻断。**

- 工作树 `package.json` / `protocol-manifest.json` 为 **`2.4.0` / `2.4`**，含 ADR-0024 `accepted`、`recordView`、`record.view.load`、response-mapping / version-negotiation / scenarios 扩展示例与 `15-v2.4-release-goals`。
- **未发现**「双重门控代码未实现」或「`fields[].key ⊆ responseMapping` 在 L2 中缺失」类机器 P0。
- **发现 2 项 P0 + 2 项 P1 + 4 项 P2 + 1 项 P3 信息簇**（`V300`–`V308`）。
- **最高权威 `01-node-protocol.md` 仍写当前协议为 `"2.3"`，且冲突时以 `01` 为准**；`page.schema.json` 描述同步陈旧——在声称发运 `2.4.0` 时属发布阻断。
- **全量** `release:check` / `test:conformance:all` / `verify:protocol-artifact` **本轮未跑**；算法 suite 静态计数为 **213**（与 `release-check.js` 期望一致）。凡依赖全量 CI 的结论标 **无法判定**。

本文档是过程记录，不是协议权威来源。权威仍以已接受 ADR、核心规范、Schema、versioned fixtures 与（打 tag 后的）协议制品 digests 为准。

### V300（P0）权威层当前版本仍为 2.3 — 已关闭

**现象：**

- `docs/01-node-protocol.md` 声明「如与其他文档冲突，以本文档为准」，却在 YAML 示意与 `meta.protocolVersion` 说明中写 **当前制品/协议使用 `"2.3"`**（约 L28、L47）；frontmatter 已是 `applies_to: schema-ui-protocol v2.4`。
- `docs/schemas/page.schema.json` `meta.protocolVersion.description` 写 **当前协议制品为 `"2.3"`**，仅展开 2.3 权限继承纪律，未提 2.4 / `recordView`。
- 对照：`package.json` / `protocol-manifest.json` / `00-overview` / CHANGELOG 为 **`2.4.0` / `2.4`**。

**风险：** 消费者与实现者按最高权威把当前 MINOR 锚在 2.3；与 2.4 制品、协商输入、迁移叙事冲突。属**原子不一致**，非措辞瑕疵。

**关闭条件：**

1. `01` 中「当前协议/制品」改为 `"2.4"` / 与 `2.4.0` 一致表述；
2. `page.schema.json` description 同步为当前制品 `"2.4"`，并点到 `recordView` / `record.view.load` 或指向迁移/L2；
3. 不依赖「仅看 frontmatter applies_to」作为关闭证据。

### V301（P0）`01` 未写 recordView 版本下限 — 已关闭

**现象：** `01` 的 `protocolVersion` 纪律列出 2.1 / 2.2 / 2.3 字段集下限，**未**写「使用 `type: recordView` 时必须 `meta.protocolVersion >= "2.4"`」。同文件 capability 列表已含 `record.view.load`（约 L49）。L2 已实现 `FLOOR_24` + capability（探针通过）。

**风险：** 冲突时以 `01` 为准 → 文档只剩 capability 一侧，**双重门控在权威层被截断**；与 ADR-0024 D1、`06`/`15`/`migration` 不一致。

**关闭条件：**

1. 在 `01` `meta.protocolVersion` 说明中增加与 2.2/2.3 同级条目：`recordView` → ≥ `"2.4"`，且须声明 `record.view.load`；
2. 与 `00` 中已有 L2 双重门控句对齐，避免只写一端。

---

### V302（P1）registry `required` 键冲突 — 已关闭

**现象：** `recordView.props.recordSource`（及历史 `form.props.recordSource`）在 DSL 源文件中先后出现布尔 `required: true|false` 与对象级 `required: ["method","url","responseMapping"]`。`JSON.parse` 后布尔被覆盖；本轮实测 `recordView.props.recordSource.required === ["method","url","responseMapping"]`。L2 `validateProps` 仅当 `spec.required === true` 时报「必填字段缺失」。缺 `recordSource` 现靠 `validateRecordSource` 特判兜底（探针通过）。

**风险：** DSL 字面「必填」与解析结果分裂；后续若删特判或改路径，必填可静默失效。

**关闭条件：**

1. 拆开字段级布尔必填与嵌套对象 `required` 数组（改名、嵌套 `schema`/`properties` 层，或 L2 同时识别数组语义且文档约定唯一写法）；
2. 缺 `recordSource` 时仍有且仅有一条稳定失败路径；
3. 可选：对 `form.recordSource` 可选性做回归，避免误改成顶层必填。

---

### V303（P1）双重门控无 L2 负例向量 — 已关闭

**现象：** `version-negotiation` 覆盖 meta 级 `record.view.load` 缺失/具备与 2.4 版本接受/拒绝，**不**在页面树上放置 `type: recordView`。conformance **无**对标权限 2.3 `permissions-inheritance` 的 L2 页负例（`2.3`+recordView → `PROTOCOL_VERSION_TOO_LOW`；`2.4` 无 capability → 缺能力）。`15` G1/G2 将 L2 双重门控勾为完成，但可发布矩阵不防 L2 回归。

**风险：** 门控仅存于脚本；重构易丢 floor 或 capability 一侧而不被 CI 抓住。

**关闭条件（择一或组合）：**

1. 增加可自动跑的 L2 负例（fixture suite、validate 金样例，或 release-check 抽样）；或
2. 在 `15`/迁移中明确「L2 负例非本 MINOR 门禁」并降级勾选措辞——**不推荐**作为默认关闭路径。

---

### V304（P2）`02` §13 未覆盖 recordView 路由 undefined — 已关闭

**现象：** `02` §11.3 已允许 form/recordView 的 recordSource 绑定；§13 仅写 `form.recordSource` 绑定键为 `undefined` 时拒绝构造请求。ADR D2 要求同构 fail-closed。

**关闭条件：** §13（或交叉引用 ADR-0021/0024）对称写入 `recordView.props.recordSource`。

---

### V305（P2）`08` 缺少 recordView 执行节 — 已关闭

**现象：** ADR 写字段与执行语义以 ADR + `02`/`03`/`08`/Schema/L2 为准；`08` 仅 §3.4 能力表一行（`form.record.load` 同样偏薄）。挂载 GET、mapping→values、节点 error、与 DataRef 边界在 `03`/ADR，未收敛到 Renderer 规范。

**关闭条件：** 在 `08` 增加简短执行节（可与 form.record.load 并列），覆盖挂载一次 GET、显式 mapping、缺路径 null、失败节点态、capability 名。

---

### V306（P2）key ⊆ mapping 仅 L2 — 已关闭

**现象：** ADR/L2/`06` 要求 `fields[].key` 必须是 `responseMapping` 键；JSON Schema/DSL 无法表达跨字段 ⊆。仅跑 L0/L1 的接入方会漏检（与历史 L2 条件语义模式一致）。

**关闭条件：** 保持 L2 为权威执行点，并在 `06`/`03` 明确「L0/L1 不足」；或增加 L2 金样例纳入 CI（可与 V303 合并）。

---

### V307（P2）场景阅读顺序遗漏 detail — 已关闭

**现象：** `docs/05-scenarios/README.md` 文件表含 `admin-list-detail-lifecycle`，推荐阅读顺序 1–9 **无**该项。

**关闭条件：** 阅读顺序补入 detail 生命周期（建议在 edit 之后）。

---

### V308（P3）信息簇 — 已关闭

| 子项 | 现象 |
|---|---|
| states | ADR D4 写 loading/error/**empty** 可选；`03` 摘要多写 loading/error |
| response-mapping 负例 | formRecord 有空 mapping 拒绝；recordView 共享实现但无对称 case |
| manifest 样例 | informative 列 `order-list-batch`，未列 `order-detail-lifecycle` |
| `06` 元数据 | `last_updated: 2026-07-23` 而 `applies_to` 已 v2.4 |

**关闭条件：** 逐项勘误或显式接受「本 MINOR 不改」并记入 checklist 决议。
