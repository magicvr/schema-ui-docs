### V6（🟡）`schemas/component-registry.json` Schema 方言声明自相矛盾

**问题：** 文件头部声明 `$schema: draft-07` 且内部使用了合法的 JSON Schema `$ref` 语法，但整体结构**完全不是合法的 draft-07 JSON Schema**：
- 顶层使用自造键 `"components"` 而非 `"properties"`/`"definitions"` 等 schema 词汇；
- 必填性写成内联布尔值 `"required": true`，而非标准的 `"required": [...]` 数组；
- `"additionalProperties": false` 被作为 `props` 对象内的普通兄弟字段，而非模式关键字。

若用 `ajv` 校验，该文件不会产生其"看起来"应有的约束效果（`ajv` 会将 `components`/内联 `required` 当成无意义属性忽略）。它与 `page.schema.json`/`node.schema.json`/`action.schema.json`/`reaction.schema.json` 四个严格遵循 JSON Schema 语法的文件形成强烈对比。

**处理方案（选其一）：**
- **方案 A（推荐）：** 将 `$schema` 改为 `"https://internal/schema-ui/component-registry-dsl#"` 或其他自定义标识，明确声明这不是标准 JSON Schema，以避免读者和工具的误解。
- **方案 B：** 维持现状，但在 `03-component-registry.md` 或文件注释中显式说明该文件的"自定义 DSL"性质及其与真正 JSON Schema 的区别。
- **方案 C：** 如果 CI 流程确实需要该文件能被标准 JSON Schema 校验器处理，则需将其重构为合法的 draft-07 Schema（较复杂，涉及结构重大调整）。

**涉及文件：** `docs/schemas/component-registry.json`、可选：`docs/03-component-registry.md`
