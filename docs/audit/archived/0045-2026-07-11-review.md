# 审计报告 0045：L3a `$self` 挂载漏检、文档/ADR 残留、MCP 场景回归与元数据卫生

## 背景

本轮基于 0044 已归档后的再次全量复审。0044 中的 value 冲突规则、表格 scope 措辞、`$row` dependencies 权威规则、表格 form-scope 禁 `required`/`value`、search/upload 场景与版本叙事均已确认落地，本轮不重复报告 V162–V174。

新问题集中在四类：

1. L3a 对 `$self` 挂载点仍不完整（表单 `visibleWhen`、表格 actions 的 form-scope）；
2. V97/V131/V164/V167/V171 落地后文档与 ADR 残留（表单字段 `scope: row` 死路径、非表单 visibleWhen 口径、`requestMapping` 的 `$` 判定、ADR-0004/0006 与 `06` 自检清单）；
3. V169 新增官方场景未进入 MCP `validate_content` 回归；
4. lock 版本、BOM、场景文案、L4 文档范围、MCP 实施计划目录树等卫生项。

扫描活跃区与归档区后，历史最大审计编号为 `0044`，本轮使用 `0045`。全局最大问题编号为 `V174`，因此本轮从 `V175` 连续编号到 `V190`。

## 发现的问题

### V175（🟡）表单上下文 `visibleWhen` 放行 `$self`

**位置：** `scripts/validate-l3a-expressions.js`（`location === 'visibleWhen'`）；`docs/01-node-protocol.md` §3.8；`docs/02-reaction-expression.md` §2 / 附录 A。

**当前值：** `01` 写表单 `visibleWhen` 仅 `$deps.*` / `$context.*`；附录 A 对 `$self` 为 ❌。L3a 仅对非表单路径做 `$context.*` 白名单；表单内 `when: "$self == true"` **通过**。

**问题：** 可通过完整校验的配置在 `visibleWhen` 上无 `$self` 注入语义，行为未定义。

**影响：** 🟡 中等。生成器/CI 无法拦截非法配置。

**建议修复：** L3a 在 `location === 'visibleWhen' && hasFormContext` 时拒绝 `$self` / `$row.*` / `$parentRow.*`（仅 `$deps.*` + `$context.user/features.*`）；补 MCP 反例。

**需要同步的文件：** `scripts/validate-l3a-expressions.js`、`mcp/tests/*`；必要时微调 `02` §9 措辞。

---

### V176（🟡）表格 `actions` 在 `scope: form` 下仍可写 `$self`

**位置：** L3a 规则 8；`02` §10.3；ADR-0004 D2b；附录 A。

**当前值：** ADR/附录 A 对 actions 任意 scope 禁 `$self`；L3a 仅 `tableAction && scope === 'row'` 时拒绝。`scope: form` + `$self` **通过**。

**问题：** 与列 form-scope 已禁 `$self` 不对称；行操作无「当前单元格」绑定对象。

**建议修复：** 规则改为任意 `tableAction` 禁 `$self`；同步 §10.3 / `08` §5.5；补反例。

**需要同步的文件：** `scripts/validate-l3a-expressions.js`、`docs/02-reaction-expression.md`、`docs/08-renderer-spec.md`、`mcp/tests/*`。

---

### V177（🟡）「表单字段 `scope: row`」死路径仍写在权威表与 Renderer 规范

**位置：** `02` §9.3 状态键表；`08` §5.5；对照 L3a `ROW_SCOPE_MOUNT`。

**当前值：** §9.3 仍有「表单字段 `scope: row`」列；08 写「表单字段上 `scope: row` 的同一限制仍适用」。L3a：`scope: row` 仅 `tableColumn` / `tableAction`。

**问题：** 读者会以为表单字段可声明 `scope: row`；实际整配置被拒绝。

**建议修复：** 删除死列；状态键表仅保留表单 form + 表格 columns/actions；08 改为引用挂载限制；§10 增加 `ROW_SCOPE_MOUNT`（可与 V190 一并）。

**需要同步的文件：** `docs/02-reaction-expression.md`、`docs/08-renderer-spec.md`。

---

### V178（🟡）`requestMapping` 字符串中 `$` 的判定：`07` vs L2/08 漂移

**位置：** `07` §3.1；`scripts/validate-l2-components.js`；`08` 行级请求段。

**当前值：** `07`「以 `$` 开头」才限 `$row.*`；L2 `includes('$')` 且非整段 `$row.*` 即拒。

**建议修复：** 重写 `07` 为「含 `$` 则必须整体为合法 `$row.*`」。

**需要同步的文件：** `docs/07-actions-contract.md`。

---

### V179（🟡）`01`/`08` 对非表单 `visibleWhen` 禁令仍停在「禁 `$deps`」

**位置：** `01` §3.8；`08` §5.5；对照 `02` §10.1 与 L3a。

**建议修复：** 改为仅允许 `$context.user.*` / `$context.features.*`，并点名禁 `$self`/`$row`/`$parentRow`。

**需要同步的文件：** `docs/01-node-protocol.md`、`docs/08-renderer-spec.md`。
