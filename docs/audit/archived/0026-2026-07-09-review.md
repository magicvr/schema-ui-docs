## 发现的问题

### V43（🔴）L3a 未按变量可见性矩阵拒绝非法 `$parentRow.*` 使用

**位置：** `docs/02-reaction-expression.md` 变量命名空间表与附录 A 变量可见性矩阵；`scripts/validate-l3a-expressions.js` 作用域隔离检查

**当前值：** `02-reaction-expression.md` 已声明：

```text
`$parentRow.<字段名>` 仅用于嵌套表格内 `scope: row` 表达式
```

附录 A 也明确：

```text
表格列 `scope: row` 表达式           `$parentRow.*` = ❌
表格列 `scope: row`（嵌套表格内）   `$parentRow.*` = ✅（仅直接父级）
表格 actions（`scope: row`）         `$parentRow.*` = ❌
表格 actions（嵌套表格内）          `$parentRow.*` = ✅（仅直接父级）
```

但 `validate-l3a-expressions.js` 只将 `$row.*` 计入 `hasRowRef`：

```javascript
const hasRowRef  = vars.some(v => v.startsWith('$row.') || v === '$row.__index' || v === '$row.__key');
```

`$parentRow.*` 虽在变量白名单中，却没有参与 `scope: form` / 非表格位置的隔离判断。

**实测证据：** 以下非法配置位于非表格、非行级位置，按当前协议应被 L3a 拒绝：

```yaml
body:
  type: section
  visibleWhen:
    when: "$parentRow.status == 'active'"
```

实际执行 `node scripts/validate-l3a-expressions.js <临时样例> --json` 返回：

```json
{
  "violations": [],
  "parseErrors": []
}
```

**问题：** 协议正文把 `$parentRow.*` 定义为严格受限的嵌套行级变量，但当前 L3a 静态校验会放行非行级位置的 `$parentRow.*`。这会造成“文档禁止、校验通过”的直接矛盾。

**影响：** 🔴 严重。非法变量可通过 CI / MCP 校验进入运行时，Renderer 在无父行上下文的位置无法一致注入 `$parentRow`，可能导致不同实现分别采用 `undefined` 降级、运行时报错或自定义兼容逻辑，破坏表达式作用域的静态可审计性。

**建议修复：** 在 L3a 中将 `$parentRow.*` 纳入行级上下文变量判断，至少拒绝 `scope: form`、非表格 `visibleWhen`、普通节点 `reactions` 中出现 `$parentRow.*`。如要完整贴合附录矩阵，还需在遍历表格列/行内操作时区分是否处于嵌套表格上下文：仅嵌套表格内的 `scope: row` 表达式允许 `$parentRow.*`。

### V44（🟢）`01-node-protocol.md` 对表达式求值时序章节的引用仍指向旧 §13

**位置：** `docs/01-node-protocol.md` §3.10；`docs/02-reaction-expression.md` §14

**当前值：** `01-node-protocol.md` 中写作：

```markdown
完整规则见 [02-reaction-expression.md §13（现 §14）](../../02-reaction-expression.md#14-表达式求值时序模型since-024)
```

但当前 `02-reaction-expression.md` 中该章节标题为：

```markdown
## 14. 表达式求值时序模型（since 0.2.4）
```

**问题：** v0.2.5 新增 `$context` 最小字段集后，`02-reaction-expression.md` 章节编号已重排，CHANGELOG 也记录“§13 求值时序重编号为 §14”，但 `01-node-protocol.md` 的章节号和锚点未同步。

**影响：** 🟢 轻微。不会改变协议语义，但会导致读者按链接跳转到不存在的旧锚点，降低核心协议文档的导航准确性。

**建议修复：** 将 `01-node-protocol.md` 中该链接更新为 `02-reaction-expression.md §14` 与 `#14-表达式求值时序模型since-024`。

## 与已归档审计的关系

- V43 与 0016 V17（Renderer 示例补充 `$self` / `$row` / `$parentRow` 注入边界）相关，但本轮不是示例覆盖不足，而是发现 L3a 实际校验未执行 `$parentRow.*` 的作用域隔离。
- V43 与 0024 V36（L3a 表格表达式默认 scope 修复）同属表达式静态校验收敛，但本轮对象是 `$parentRow.*`，不重复 `$row.*` 默认 scope 问题。
- V44 与 0020 V26 / V27、0021 V28 属于章节重排后的交叉引用漂移，本轮定位的是 `01-node-protocol.md` 指向 `02-reaction-expression.md` 求值时序章节的旧 §13 锚点。

## 建议处理顺序

1. 优先修复 V43。它会导致协议禁止的表达式通过静态校验，属于正文与工具链的直接冲突。
2. 修复 V44。该项为单行文档链接修正，可与 V43 同轮完成。

## 防止复发建议

- L3a 对变量命名空间新增或扩展时，应同步维护“白名单”和“作用域隔离变量集合”，避免只允许解析、不执行语义隔离。
- 将 `02-reaction-expression.md` 附录 A 的变量可见性矩阵转化为 L3a 回归样例，至少覆盖 `$deps`、`$row`、`$parentRow`、`$self` 的正反例。
- 对带章节编号的跨文档链接增加轻量检查，尤其是 `02-reaction-expression.md` 这类频繁插入新章节的规范文档。
