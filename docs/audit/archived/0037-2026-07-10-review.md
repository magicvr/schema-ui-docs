## 发现的问题

### V96（🔴）表格表达式的 `scope: form` 合法性在核心协议与组件正文中冲突

**位置：** `docs/01-node-protocol.md` §3.5；`docs/03-component-registry.md` table 作用域说明；ADR-0004 D2/D3。

**当前值：** `01` 将表格 `columns[]` / `actions[]` 概括为“必须显式 `scope: row`”；`03` 与 ADR-0004 则允许表单内表格使用默认 `scope: form` + `$deps.*`，仅在使用 `$row.*` 时要求显式 `scope: row`。

**问题与影响：** 核心协议把合法的表单级表格表达式描述成非法。生成器、Renderer 和静态校验器若分别以不同正文为准，会形成互不兼容的实现。

**建议修复：** 将 `01` 改为“使用 `$row.*` / `$parentRow.*` 或行级 `$self` 时必须显式 `scope: row`；表格位于 form 上下文时可使用 `scope: form` + `$deps.*`”，并同步总纲中同类概括。

**需要同步：** `docs/00-overview.md`、`docs/01-node-protocol.md`，必要时校对 `docs/03-component-registry.md` 与 ADR-0004。

### V97（🔴）L3a 未限制 `scope: row` 的挂载位置，也不校验 `$row.*` 依赖声明

**位置：** `scripts/validate-l3a-expressions.js` 的 `validateExpression()` / `scanNode()`；`docs/schemas/reaction.schema.json`；ADR-0004 D1/D2。

**当前值：** 普通 `input.reactions` 可声明 `scope: row`；表格行级表达式使用 `$row.status` 时，即使 `dependencies: []` 也不会报告未声明依赖。两种反例均实测通过 L0-L4/MCP。

**问题与影响：** `$row` 只应在表格列/行内操作中注入，且 `scope: row` 的 `dependencies` 应列出行字段路径。当前实现会接受运行时没有行上下文或响应式依赖不完整的配置。

**建议修复：** 按表达式挂载位置限制 `scope: row`；提取 `$row.*` 与 `$parentRow.*` 路径并核对 `dependencies`；补充普通 Node 误用、缺失行依赖及合法行级正例。

**需要同步：** `scripts/validate-l3a-expressions.js`、`docs/06-validation.md`、`mcp/tests/test-utils.ts`、`mcp/tests/validate-content.test.ts`。

### V98（🔴）空 `when` 与畸形变量点路径可绕过表达式校验

**位置：** `docs/schemas/reaction.schema.json`、`docs/schemas/node.schema.json` VisibleWhen；`scripts/validate-l3a-expressions.js` tokenizer 与各 `if (...when)` 调用点。

**当前值：** Schema 只要求 `when` 为 string，无 `minLength`；L3a 使用真值判断，导致 `when: ""` 不进入已有“表达式不能为空”检查。tokenizer 又将连续点视为变量字符，`$row..status`、`$row.` 等畸形路径可作为 VAR 通过。

**问题与影响：** 语法无效的条件配置可通过完整校验链，Renderer 只能自行决定空条件与畸形变量的行为。

**建议修复：** Schema 为 reaction/VisibleWhen 的 `when` 增加 `minLength: 1`；L3a 按字段是否存在调用 `validateExpression()`；对所有变量命名空间使用完整锚定的点路径正则。

**需要同步：** `docs/schemas/reaction.schema.json`、`docs/schemas/node.schema.json`、`scripts/validate-l3a-expressions.js`、`docs/06-validation.md`、MCP 测试。

### V99（🔴）L2 忽略组件 DSL 嵌套字段上的布尔 `required: true`

**位置：** `docs/schemas/component-registry.json` table `pagination.mode`；`scripts/validate-l2-components.js` `validateNestedObject()`。

**当前值：** DSL 用字段级 `required: true` 表示 `pagination.mode` 必填，但递归校验器只读取数组形式的 `spec.required`。`pagination: {}` 实测通过 L2/MCP。

**问题与影响：** 组件注册表的机器约束没有被校验器忠实执行，任何嵌套字段级必填项都可能被静默忽略。

**建议修复：** `validateNestedObject()` 同时执行子字段规格上的 `required: true`；增加 `pagination: {}` 反例与合法分页正例，并扫描 DSL 中其他嵌套布尔必填字段。

**需要同步：** `scripts/validate-l2-components.js`、`docs/06-validation.md`、MCP 测试。

### V100（🔴）行级 `requestMapping` 未执行模板禁令和 `$parentRow` 嵌套边界

**位置：** `scripts/validate-l2-components.js` `validateRequestMappingValues()` / `validateRowActionRefs()`；ADR-0008 D3；`docs/07-actions-contract.md` §3.1。

**当前值：** 仅当字符串以 `$` 开头时才校验引用，因此 `prefix-$row.id` 被当作普通字面量；正则同时接受 `$row.*` 与 `$parentRow.*`，但遍历未传递表格深度，顶层表格也可使用 `$parentRow.id`。两类反例实测通过。

**问题与影响：** 明确禁止的模板字符串和无父行上下文的引用进入运行时，破坏行级请求映射的“纯字段绑定”边界。

**建议修复：** 含 `$` 的字符串必须整体匹配合法行引用；扫描 RowAction 时携带表格深度，仅嵌套表格允许 `$parentRow.*`；补充 path/query/body 三段正反例。

**需要同步：** `scripts/validate-l2-components.js`、`docs/06-validation.md`、MCP 测试。

### V101（🔴）顶层 `datasources` 可形成未定义的引用链、自引用或循环引用

**位置：** `docs/schemas/page.schema.json` datasources；`docs/schemas/node.schema.json` DataRef；`docs/04-datasource-contract.md` §1；`docs/08-renderer-spec.md` §2.1.1。

**当前值：** `datasources.*` 复用完整 DataRef，因此可声明 `source: ref`；Renderer 规格只定义 Node 引用 datasource 后直接读取请求配置，没有 datasource 链式解析、终止条件或循环处理规则。自引用实测通过 L0-L4/MCP。

**问题与影响：** 同一配置可能被实现为递归解析、单层读取失败或直接拒绝，严重影响跨 Renderer 一致性，并存在无限递归风险。

**建议修复：** 优先将顶层 datasource 限定为 `source: api` / `source: static`，禁止 `source: ref`；若确需引用链，则必须先用 ADR 定义存在性、终止与循环检测。

**需要同步：** `docs/schemas/page.schema.json`（可新增专用 datasource definition）、`docs/01-node-protocol.md`、`docs/04-datasource-contract.md`、`docs/08-renderer-spec.md`、L2/MCP 测试。

### V102（🟡）重复 Node `id` 在 L2 中被静默覆盖

**位置：** `docs/01-node-protocol.md` §3.6；`docs/schemas/node.schema.json` id 描述；`scripts/validate-l2-components.js` `validateDataRefsAndTargetTable()`。

**当前值：** 协议要求同一页面内唯一并报错；L2 收集 `nodeIds` 时直接覆盖已有条目，两个合法 Node 使用相同 id 实测通过。

**问题与影响：** 重复 id 会令定位、patch、埋点和 `targetTable` 引用产生歧义，且当前引用校验可能绑定到最后一个节点。

**建议修复：** 收集 id 时立即报告首个路径与重复路径，不覆盖原条目；遍历范围保持覆盖 body、tabs content 与 modal content。

**需要同步：** `scripts/validate-l2-components.js`、`docs/06-validation.md`、MCP 测试。

### V103（🟡）`request.bodyMapping` 缺少值类型约束

**位置：** `docs/07-actions-contract.md` §3；`docs/schemas/action.schema.json` RequestAction.bodyMapping。

**当前值：** 正文定义为“表单字段名 -> 请求体字段名”，但 Schema 仅声明自由 object，嵌套对象、数组和数字均可通过。

**问题与影响：** Renderer 无法确定目标字段名的类型，错误配置只能在提交时暴露。

**建议修复：** 将 `bodyMapping.additionalProperties` 限定为非空 string，并补充 L1/MCP 正反例；正文明确 key/value 方向和空字符串边界。

**需要同步：** `docs/schemas/action.schema.json`、`docs/07-actions-contract.md`、MCP 测试。

### V105（🟡）正文错误地用 `dependencies` 是否存在推断 `visibleWhen` 上下文

**位置：** `docs/02-reaction-expression.md` §10.1；ADR-0003 D3a；对照 `docs/01-node-protocol.md` §3.8 与 L3a `isFormContext()`。

**当前值：** 两处写“未声明 `dependencies`（即处于非表单上下文）”。正式规则实际按节点是否位于 form 树中判断；表单内缺少 dependencies 是错误配置，不会因此变成非表单上下文。

**问题与影响：** Renderer 或第三方校验器可能从字段缺失反推作用域，错误接受表单内不完整配置。

**建议修复：** 将因果关系改为“节点处于非表单上下文时 dependencies 可省略”；明确表单上下文由 Node 树位置决定。

**需要同步：** `docs/02-reaction-expression.md`、`docs/decisions/0003-context-namespace-and-visible-when.md`。

### V106（🟡）ADR-0006 求值时序示例依赖不存在的跨字段写入语义

**位置：** `docs/decisions/0006-expression-evaluation-order.md` 背景示例与 D3。

**当前值：** `country` 字段自身 reaction 的 `fulfill.value: null` 被注释为“清空 province”，但协议没有 target 字段，`value` 只作用于当前字段。D3 又讨论“不同节点写入同一目标字段”，同样缺少可表达语法。

**问题与影响：** 用于定义稳定快照的核心 ADR 建立在不可表达行为上，可能误导 Renderer 实现跨字段写入扩展。

**建议修复：** 用两个合法 reaction 对同一节点状态/值产生确定性覆盖的场景重写示例；删除或改写 D3 中不存在的跨节点目标字段写入描述。

**需要同步：** `docs/decisions/0006-expression-evaluation-order.md`，并校对 `docs/08-renderer-spec.md` 求值时序描述。

### V108（🟡）MCP `get_doc` 的 20KB 限制按字符数而非 UTF-8 字节数计算

**位置：** `mcp/src/core/protocol-index.ts` `getDoc()`；`docs/mcp/v1-design.md` §3.2；ADR-0007。

**当前值：** 实现用 `content.length > 20 * 1024`。实测 component-registry 为 18,447 字符/26,830 UTF-8 字节，renderer-spec 为 16,035 字符/24,424 字节，两者均返回 `truncated: false`。

**问题与影响：** 中文文档显著超过约定的单次返回体积却不截断，也不会提供 `availableSections`。

**建议修复：** 使用 `Buffer.byteLength(content, 'utf8')` 判断，并按 UTF-8 字节边界截断，确保提示文本也计入上限；补充多字节内容测试。

**需要同步：** `mcp/src/core/protocol-index.ts`、`mcp/tests/search.test.ts`，必要时细化 `docs/mcp/v1-design.md` 的 KB 定义。

### V109（🟡）MCP 搜索索引遗漏首个二级标题之前的标题、frontmatter 与导言

**位置：** `mcp/src/core/protocol-index.ts` `splitSections()`；`mcp/src/core/search.ts` `rankSearchResults()`。

**当前值：** 文档存在 `##` 时，`splitSections()` 只返回从第一个 `##` 开始的区段。实测搜索只出现在 reaction-expression 导言中的“唯一涉及 类代码”返回空结果。

**建议修复：** 将首个 `##` 前的非空内容作为稳定的“导言”区段，并避免 H1 标题重复计分；补充导言关键词和排序测试。

**需要同步：** `mcp/src/core/protocol-index.ts`、`mcp/src/core/search.ts`（若排序需调整）、`mcp/tests/search.test.ts`。

### V110（🟡）MCP 分层违规泄露内部临时路径且忽略调用方 `filename`

**位置：** `mcp/src/core/validation-runner.ts` `runLayerScript()` / `mapScriptViolation()` / `firstParseError()`；ADR-0007 D3/D5。

**当前值：** 只有 parseError 会覆盖调用方 filename；L2/L3a/L4 violation 保留子脚本看到的 UUID 临时文件路径。传入 `filename: caller/page.json` 时，L2 实测返回 `os.tmpdir()/schema-ui-mcp-.../page-UUID.json`。

**问题与影响：** 错误展示不符合 filename 的上下文提示语义，结果不稳定，并向远程调用方暴露服务端目录结构。

**建议修复：** 统一把分层违规的 `file` 映射为调用方 filename；未提供 filename 时省略 `file`，不得返回内部路径；覆盖 L2/L3a/L4 测试。

**需要同步：** `mcp/src/core/validation-runner.ts`、`mcp/tests/validate-content.test.ts`，必要时在 MCP 设计中明确省略规则。
