# 审计报告 0064：2.1/2.2 协议纪律补强

## 结论

在 0062/0063 互操作闭合之上，对版本纪律与文档对称性做轻量复审并**当场整改**。

**未发现 P0。** **1 项 P1 + 4 项 P2**（`V282`–`V286`）**全部关闭**。

本文档是整改记录，不是协议权威来源。权威语义以 ADR-0020/0021/0022、`01`/`02`/`06`/`07`/`08`、L2 与 migrations 为准。

## 问题清单

| 编号 | 级别 | 主题 | 状态 |
|---|---|---|---|
| V282 | P1 | 字段集→`protocolVersion` 下限无 L2 门禁 | **已关闭** |
| V283 | P2 | page Trigger navigate 未绑定模板 L2 不对称 | **已关闭** |
| V284 | P2 | 版本/能力叙事漂移（schema/01/00/11） | **已关闭** |
| V285 | P2 | `$selection` 与 batch 检查清单文档缺口 | **已关闭** |
| V286 | P2 | page Trigger「静态 body」文案误导 | **已关闭** |

---

### V282（P1）— 已关闭

**裁决：** L2 增加 `validateProtocolVersionFloor`：

| 字段 | 最低 `protocolVersion` |
|---|---|
| `actionButton` / `table.toolbar` / `navigateMapping` / `form.recordSource` | `"2.1"` |
| `table.selection` / `requiresSelection` / `batchMapping` | 过渡期 `"2.1"`（`ALLOW_22_FIELDS_ON_21=true`）；`2.2.0` tag 后 `"2.2"` |

错误文案含 `PROTOCOL_VERSION_TOO_LOW`。负例：`_samples/audit-0064-v21-fields-on-20-bad.yaml`。同步 migrations、ADR-0022 状态段、`13` G1。

### V283（P2）— 已关闭

`validateActionTrigger` 对 page Trigger `navigate` url 与 request 同样拒绝未绑定 `{`/`}`。负例：`audit-0064-page-trigger-navigate-template-bad.yaml`。

### V284（P2）— 已关闭

`page.schema.json`、`01` YAML 注释、`00` capability 列表、`11` Phase B 勾选句与 2.1 制品 / 迁移纪律对齐。

### V285（P2）— 已关闭

`02` §2 增加 `$selection.keys` / `$selection.count` 与禁止事项；`06` 自检清单补 selection/batch/版本下限。

### V286（P2）— 已关闭

ADR-0020 D4 与 `07` §3.3：page Trigger request **body 恒 null**；RequestAction 无静态 body 字段。

## 关闭证据

- L2 负例：`audit-0064-v21-fields-on-20-bad.yaml`、`audit-0064-page-trigger-navigate-template-bad.yaml` 按期望失败。
- 正例样例（含 `order-list-batch` / lifecycle）L2 通过。
- `npm run validate:scenarios` / `check:links` / `test:conformance:all` / `release:check`（以关闭时工作区为准）。
