# 审计报告 0062：协议 2.1 与 2.2 互操作性复审

## 结论

本轮未发现 P0 问题，并确认 capability 门控的 ADR-0022 页面在正式 2.2 制品发布前可继续声明 `meta.protocolVersion: "2.1"`。现有版本协商模型支持该发布策略。

规范 ADR/Schema/L2 规则、JavaScript/Python 参考实现与 conformance 向量之间曾有 6 项互操作性缺口（P1 × 5、P2 × 1）。**V267–V272 现已全部完成协议裁决与落地：** 路径模板 fail-closed、toolbar L3a 上下文、batch 选择前置、recordSource method 必填、selection 键规范化、以及 page Trigger navigate/modal/confirm 可观测结果均已同步到 ADR/规范、L2/L3a、双端 reference 与 fixtures。

本文档是整改记录，不是协议权威来源。权威语义以 ADR-0020/0021/0022、`03`/`07`/`08`、component DSL、校验器与 conformance fixtures 为准。

## 范围与基线

纳入范围：

- 随协议 2.1 发布的 ADR-0020 与 ADR-0021 新增内容；
- 目标为 2.2 的 ADR-0022 表格选择与批量 request 新增内容；
- 组件注册 DSL、L2/L3a 校验、JS/Python 参考实现、fixtures、版本协商和官方示例。

明确排除：

- 生产 Renderer、Host UI、业务后端与授权实现；
- 视觉行为、DOM、CSS 和客户端路由库细节；
- capability/版本契约以外的协议发布打包实现。

### V267（P1）参考请求构造接受未解析的 URL 模板 — 已修复

**证据：** L2 在 [`validate-l2-components.js`](../../../scripts/validate-l2-components.js) 中正确要求 URL 占位符与 `path` mapping 键匹配，但 reference 曾只替换 mapping 提供的键。

**整改：** `applyPathParams`（JS/Python）在替换前校验占位符集合与 path 键集合完全相等：`MISSING_PATH_BINDING` / `EXTRA_PATH_BINDING` / `INVALID_URL_TEMPLATE` / `UNRESOLVED_PATH_TEMPLATE`。覆盖 row navigate、row action、recordSource、batch path。fixtures 已增负向与合法 path 字面量向量。同步 `07` / `08` / ADR-0021 / ADR-0022。

### V268（P1）L3a 未校验 toolbar 表达式 — 已修复

**整改：** [`validate-l3a-expressions.js`](../../../scripts/validate-l3a-expressions.js) 遍历 `table.props.toolbar[]` 的 `visibleWhen` 与 `permissions`；toolbar 强制 `hasFormContext: false`，仅允许 `$context.*`。样例：`docs/05-scenarios/_samples/audit-0062-toolbar-*.yaml`。同步 ADR-0020 D5 与 `06`。

### V269（P1）`batchMapping` 未要求表格声明选择模型 — 已修复

**整改：** L2 在 `batchMapping` 存在时要求同 table `props.selection.mode: multiple`。**裁决 OQ-22-6：`requiresSelection: true` 对 batch 不强制**（运行时仍有 `EMPTY_SELECTION`；推荐示例保留 `requiresSelection: true`）。样例：`audit-0062-batch-without-selection-bad.yaml`。同步 ADR-0022 / `03` / `07`。

### V270（P1）参考实现为 `recordSource.method` 提供默认值 — 已修复

**整改：** 缺失 method → `MISSING_RECORD_SOURCE_METHOD`（`recordSource.method`）；保留 `RECORD_SOURCE_METHOD_NOT_GET`。fixture：`record-source-missing-method-rejected`。同步 ADR-0021 / `03`。

### V271（P1）选择状态未强制标量和唯一键 — 已修复

**整改：** table-query-state（JS/Python）规范化 keys：仅 string / finite number / boolean；去重保序；`count = keys.length`。fixtures：`selection-dedupes-keys-preserve-order`、`selection-filters-non-scalar-keys`、`selection-mixed-scalar-types-and-count`。同步 ADR-0022 D2。

### V272（P2）ActionTrigger navigate/modal 缺少 conformance 覆盖 — 已修复

**整改：** request-construction 新增 `pageTriggerNavigate` / `pageTriggerModal`，并统一 `confirm` / `confirmAccepted` 门控（`CONFIRM_REJECTED`）覆盖 request/navigate/modal/batch。可观测结果：`navigation.url`、`modalOpen`、`request`。同步 ADR-0020 / `07` / conformance README。

## 已验证的非问题

源 package 仍为 artifact/protocol 版本 2.1，而 ADR-0022 标记为未来 2.2 制品的一部分。这是当前 Unreleased 轨道的有意设计：在正式 2.2 制品发布前，ADR-0022 允许页面使用协议版本 2.1 加上 `table.selection` 与 `actions.batch.request` capability。版本协商 fixture 已覆盖缺失 capability 的拒绝行为，因此不将其记录为发布版本缺陷。
