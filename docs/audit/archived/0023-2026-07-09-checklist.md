## V32（🔴）`validate-l2-components.js` `getDeclaredFields()` 的 `reserved` 集合误杀表单组件 `required` 业务字段

| 字段 | 值 |
|---|---|
| **位置** | `scripts/validate-l2-components.js` `getDeclaredFields()` |
| **问题** | `reserved` 集合包含 `required`，导致 6 个表单组件（`input`/`inputNumber`/`select`/`datePicker`/`dateRangePicker`/`upload`）的 `props.required`（布尔业务字段）被从已声明字段列表中排除，`additionalProperties: false` 检查判为非法 |
| **操作** | 将 `required` 从 `reserved` 集合中移除。`validateProps()` 中的必填检查通过 `spec.required === true` 处理，`validateNestedObject()` 中通过 `Array.isArray(spec.required)` 处理数组风格 required，均不依赖 `getDeclaredFields` 的过滤 |
| **文件** | `scripts/validate-l2-components.js` |

- [x] V32a 将 `reserved` 集合中的 `'required'` 移除，由：
  ```js
  const reserved = new Set(['additionalProperties', 'required', 'allOf', 'anyOf', 'oneOf']);
  ```
  改为：
  ```js
  const reserved = new Set(['additionalProperties', 'allOf', 'anyOf', 'oneOf']);
  ```
- [x] V32b 用构造的表单页面样例执行 `node scripts/validate-l2-components.js`，验证 `input.props.required: true` 不再被误报
- [x] V32c 回归测试其他表单组件（`inputNumber`/`select`/`datePicker`/`dateRangePicker`/`upload`）的 `required` 字段，均已通过校验

## V33（🟡）`validateResponseMapping()` 服务端分页检查误报"未声明 responseMapping"的合法默认用法

| 字段 | 值 |
|---|---|
| **位置** | `scripts/validate-l2-components.js` `validateResponseMapping()` |
| **问题** | 服务端分页检查 `if (!rm || !rm.total)` 中的 `!rm` 会误报协议允许的"未声明 responseMapping，走默认 `total` 字段名"的合法场景 |
| **操作** | 将条件改为 `rm !== undefined && !rm.total`，与同函数中 `rm.list` 的判断模式保持一致 |
| **文件** | `scripts/validate-l2-components.js` |

- [x] V33a 将服务端分页检查条件从 `!rm || !rm.total` 改为 `rm !== undefined && !rm.total`
- [x] V33b 用构造的 server 分页页面样例（未声明 `responseMapping`）验证不再被误报
- [x] V33c 构造一个"声明了 `responseMapping` 但缺 `total`"的测试用例，验证仍能正确报错
