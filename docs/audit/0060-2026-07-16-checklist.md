## V236：DataRef 只读边界

- [x] DataRef 和页面级 API datasource 只允许 `GET`，缺省 method 按 `GET` 处理。
- [x] `params` 只进入 query；DataRef 不隐式产生请求体。
- [x] Schema、L2、JS/Python reference 对非 GET 返回同一 `DATA_REF_METHOD_NOT_READ_ONLY`。
- [x] body-based read 未被伪装成旧 DataRef 扩展，必须走独立 ADR。

## V237：Action 重试与未知结果

- [x] `retryPolicy` 只允许 `never | idempotent`，缺省值为 `never`。
- [x] `idempotent` 重试复用同一 invocation id 和 `Idempotency-Key`，页面不能配置静态 key。
- [x] timeout/network interruption 返回 `unknown`，Abort 与 HTTP 业务错误保持独立语义。
- [x] request、upload、action-outcome 的 JS/Python reference 和 fixtures 对齐。

## V238：RowAction 标量边界

- [x] `$row.*` 解析结果限制为 string、finite number、boolean 或 null。
- [x] row 字段按 own-property 读取，缺失字段统一拒绝。
- [x] `__proto__`、`prototype`、`constructor` 路径在 L2、JS/Python reference 和 fixtures 中统一拒绝。

## V239：DataRef body 语义

- [x] 明确采用 query-compatible read API，不在 DataRef 中增加隐式 body。
- [x] 需要 JSON body 的读取接口必须新增字段、方法集合、序列化规则、ADR 和 fixtures。

## V240：协议 URL grammar

- [x] 请求、上传、选项加载和导航 URL 统一为 baseURL 下的单斜杠相对路径。
- [x] 绝对 URL、协议相对 URL、空白和反斜杠形式由 Schema、L2 和 reference 统一拒绝。
- [x] URL 规则覆盖 DataRef、页面 datasource、optionsSource、request/upload action 和 navigate outcome。
