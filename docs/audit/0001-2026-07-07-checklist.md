## 🔴 P0 — 必须修复（阻塞 v0.2 发布）

- [ ] **JSON Schema CSS 禁用词校验逻辑修复**（§2.2）
  - `node.schema.json` 中 `not: { required: [...] }` 语义错误
  - 改为 `anyOf` 嵌套 `required`，或移至自定义 lint 脚本
- [ ] **`tabs` 组件 children 匹配机制补充**（§2.4）
  - 协议层面缺少 `key` 字段，无法建立 tab → children 映射
  - 推荐方案 A：`tabs.props.items` 内嵌 children；或方案 B：为 Node 增加 `key` 字段

## 🟠 P1 — 高优先级（建议 v0.2 一并解决）

- [ ] **`valueField` 从 `DataRef` 移至 `props`**（§2.1）
  - 涉及文件：`node.schema.json`、`03-component-registry.md`、场景示例
  - 保持与 `table.columns[].field`、`chart.xField/yField` 取值方式一致
- [ ] **`actions` 完整契约补充**（§2.5）
  - 补充 action 类型枚举（request / navigate / modal / custom）
  - 定义 `onSuccess` / `onError` 行为描述及 `bodyMapping` 规范
- [ ] **Node 增加 `id` 字段**（§3.1）
  - 可选 `id: string`，页面内唯一标识
  - 涉及文件：`node.schema.json`、`01-node-protocol.md`

## 🟡 P2 — 中优先级（v0.3 规划）

- [ ] **节点级条件渲染 `visibleWhen`**（§2.3）
  - 非表单节点（section、statCard、chart 等）无显隐能力
  - 需引入 `$context` / `$data` 命名空间
- [ ] **权限控制 `permissions` 字段**（§3.2）
  - Node 级别和 action 级别支持 `requireAny` / `requireAll`
  - Renderer 渲染前按当前用户权限过滤
- [ ] **`span` 提升为通用 props**（§3.7）
  - 当前仅 `section` 支持 `span`，其他 grid 子节点需额外包裹
