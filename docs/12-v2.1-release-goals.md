# v2.1 发布目标与门禁

本文档记录 **v2.0 → v2.1** 的 MINOR 发布：Admin 生命周期 P0（页面级动作入口、行级导航、表单记录加载回填）以 **capability 门控** 方式进入核心协议。未声明新 capability 的合法 `2.0` 页面语义不变；声明新能力的页面应使用 `meta.protocolVersion: "2.1"`。

权威 ADR：[0020](./decisions/0020-page-action-trigger.md)、[0021](./decisions/0021-record-navigation-and-form-load.md)。轨道背景见 [11-next-admin-lifecycle-goals.md](./11-next-admin-lifecycle-goals.md)。迁移见 [migrations/2.0-to-2.1.md](./migrations/2.0-to-2.1.md)。

## 1. v2.0 到 v2.1 的边界

- v2.1 页面声明 `meta.protocolVersion: "2.1"`。
- Renderer 通过精确 `supportedVersions` 协商；支持 `2.1` 的 Renderer 可同时列出 `2.0` 以继续渲染旧页面。
- 新增预定义 capability：`actions.page.trigger`、`actions.row.navigate`、`form.record.load`（既有 `actions.upload`、`actions.row.request` 不变）。
- 新增组件 / 字段：`actionButton`、`table.props.toolbar`、`RowAction.navigateMapping`、`form.props.recordSource`、`$context.route`。
- 不引入批量选择、权限继承、`recordView`（仍属后续轨道）。

## 2. v2.1 阻断门禁

### G1. 版本与 capability

- [x] 协议制品 `2.1.0` / 页面协议 `2.1` 与 manifest、package 对齐。
- [x] version-negotiation 覆盖 `2.1` 接受与 Admin lifecycle capability 缺失序。
- [x] 未声明 capability 的页面不得静默启用新字段（L2 强制声明）。

### G2. 结构与静态校验

- [x] Schema / 组件 DSL 含 `actionButton`、`toolbar`、`navigateMapping`、`recordSource`。
- [x] L2 覆盖 Trigger 类型白名单、GET 禁止、mapping 互斥、recordSource 必填 mapping、search 禁止 recordSource。
- [x] 核心规范 `applies_to: schema-ui-protocol v2.1`。

### G3. 可观测算法与场景

- [x] request-construction：`rowNavigate` / `recordSource` / `pageTriggerRequest` 双端 fixtures。
- [x] response-mapping：`formRecord` 回填双端 fixtures。
- [x] scenarios：列表导航 + 编辑加载提交步进（`admin-list-edit-lifecycle`）。
- [x] 官方六场景 `protocolVersion: "2.1"` 且 L0–L4 通过。

### G4. 发布制品

- [x] `release:check` fixture digest 与协议制品 digests 可复现。
- [x] 迁移文档 `2.0-to-2.1` 与 CHANGELOG `v2.1.0` 已发布。
- [x] 协议 tar.gz 构建与 verify 通过。

## 3. 发布工程门禁

- [x] `package.json` / lockfile / `protocol-manifest.json` 统一为 `2.1.0` / `2.1`。
- [x] 算法类 fixtures `protocolVersion` 为 `"2.1"`（version-negotiation 历史向量保留）。
- [x] 链接检查、官方场景校验、全量 conformance 通过。

## 4. 版本纪律

v2.1 之后继续遵守 SemVer：文档勘误为 PATCH；兼容字段 / capability 为 MINOR；破坏合法输入或默认可观测结果为 MAJOR。批量操作等 P1 能力另开 MINOR/ADR，不回写本文件门禁。

## 5. 完成定义

G1–G4 与发布工程门禁全部关闭，且 `npm run release:check` 与 `npm run verify:protocol-artifact` 通过，即可将 `2.1.0` 视为协议制品完成。v2.0 历史门禁保留在 [10-v2-release-goals.md](./10-v2-release-goals.md)。
